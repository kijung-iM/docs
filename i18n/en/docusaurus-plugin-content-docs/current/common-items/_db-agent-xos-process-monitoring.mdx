## Process Monitoring{#xos}

To monitor the process usage of the database server, run a separate XOS agent process on the database server to collect data. You can also check the CPU usage of the active session.

1.  Set the following options in the _whatap.conf_ file on the agent server.

```ini title='whatap.conf'
xos=[DBX_DB_SERVER_IP]
xos_port=[DBX_PORT] (default: 3002)
```

  :::note
If the IP address of the DB server cannot be specified, set as `xos=1`.
  :::

1.  Move the XOS path to the database server. _Uncompressed folder/whatap/xos/_

2.  Enter the agent server data in the _xos.conf_ file on the database server.

```ini title='xos.conf'
dbx_ip=[DBX_AGENT_SERVER_IP]
dbx_port=[DBX_PORT] (default: 3002)
cpu_limit=0
mem_limit=10240
```

1.  Execute the following process on the database server.

```bash title='start.sh'
$ ./start.sh
```

```bash title='백그라운드로 실행할 경우'
$ ./startd.sh
```
