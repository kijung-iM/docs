Release date: 2022-11-29

:::caution
The Windows monitoring agent can stop around 2:00 am due to an automatic restart error and cause false detection as Agent NoData. If the version 2.2.8 is being used, it is recommended to update to the version 2.2.9 or later. You can restart the server after downloading the installation file. You do not need to delete the agent.
:::

## Enhancement in handling the Windows process monitoring owner name

-   The Windows process monitoring owner name lookup API has been improved to reduce the call load through caching.

## Enhancement in handling the Linux CPU IRQ collection error

-   When collecting Linux CPU IRQs, the under-collecting error due to a code error when calculating IRQ has been fixed.
