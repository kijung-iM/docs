<div class ='faq'>

### 로그 원문 복원

| Q. 로그를 수집한 후 원문을 복원할 수 있나요? |
| --------------------------- |

> A. 로그 원문 복원 기능을 제공하지 않습니다. 現在のログ収集は、セキュリティ目的ではなく分析目的で提供されています。
>
> :::note
> **설치형**의 경우 로그 원문 복원 툴을 별도로 제공합니다.
> :::

<br/>

### ログのバックアップと復旧

| Q. ログをバックアップして復旧できますか？ |
| ---------------------- |

> A. 수집된 로그는 일 단위로 저장됩니다. 과거 일자의 파일을 백업해 두었다면 해당 폴더에 다시 복구할 수 있습니다. 

<br/>

### 로그 데이터 삭제 주기

| Q. 로그 데이터 삭제 주기는 어떻게 되나요? |
| ------------------------- |

> A. **_로그 설정_** 메뉴에서 설정한 **_데이터 유지 기간_**과 같습니다. 다만 적재된 로그 잔존 주기는 시간 단위이기에 설정한 **_데이터 유지 기간_**에 추가로 1시간까지 데이터가 남아있을 수 있습니다.
>
> :::note
>
> **_데이터 유지 기간_**을 설정하지 않은 경우 기본값은 1일입니다. **_데이터 유지 기간_** 설정에 대한 자세한 내용은 [다음 문서](../log/log-setting#log-data-setting)를 참조하세요.
>
> :::

<br/>

### エージェントのログ転送周期

| Q. 에이전트로부터 로그 전송 주기는 어떻게 되나요? |
| ----------------------------- |

> A. エージェントのログバッファ容量(64 KB)に到達したり、転送周期(2秒)に達した場合は、ログを転送します。 네트워크 전송 시 _zip_형식으로 압축해 전송하고 수집 서버에서 수신 시 압축을 해제해 저장합니다.

<br/>

### 他社ログソリューションとの連携

| Q. 他のログソリューションと連携できますか？ |
| ----------------------- |

> A. 現在、他社ログソリューションとの連携機能は提供しておりません。
