## Preview-1.97.0

2023年3月8日

### 「共通」

#### User Interface（UI）

ユーザーの満足度を高めるために、次のようにユーザーインターフェースデザインを改善します。

-   <Status>Change</Status> <UI>利用履歴</UI> > <UI>請求書プレビュー</UI> メニューのUI改善

    -   プロジェクト、製品別にグループ化された請求書プレビュー情報の提供

-   <Status>Change</Status> 画面上段に位置する「メニューパス」を「メニュー名」として表示するように変更

    -   変更前：上位パス > メニュー名
    -   変更後：メニュー名

-   <Status>Change</Status> 画面右上のWhaTap技術文書のリンクボタン追加およびチャンネルトーク内の技術文書リンクを除去

-   <Status>Fixed</Status> サービスロゴの横のブラウザー時間の不一致通知アイコンの位置を修正

-   <Status>Fixed</Status> プロジェクトのないグループを選択すると、プロジェクト作成ページに移動しないように修正

#### Flexボード

-   <Status>Change</Status> ダッシュボードと一覧を画面に読み込む時に、読み込みインジケーターを適用

-   <Status>Feature</Status> Seriesウィジェットタイプのチャート設定で凡例表示を追加

-   <Status>Feature</Status> Seriesウィジェットタイプに累積チャートタイプを追加

-   <Status>Feature</Status> Seriesウィジェットタイプのチャート設定でチャート色基準のオプションを追加

    -   従来：タグを基準にチャートの色を表現
    -   追加：フィールドを基準にチャートの色を表示するオプション

-   <Status>Feature</Status> DB製品群で次のウィジェットテンプレートを追加

    -   **DB共通**
        -   CPU %（xos）ウィジェット
        -   Memory %（xos）ウィジェット
    -   **MySQL**
        -   スロークエリ
    -   **PostgreSQL**
        -   スロークエリ
        -   PostgreSQL Wait Event件数ウィジェット
        -   PostgreSQL Client Ip件数ウィジェット
        -   PostgreSQL統計テーブル
        -   PostgreSQLアクティブセッション件数
        -   PostgreSQLアクティブセッションテーブル

#### ログ

-   <Status>Change</Status> <UI>ログ</UI> メニューの機能を容易に理解するため、メニュー名を変更

    -   <UI>ライブTail</UI> → <UI>ライブテール</UI>
    -   <UI>探索機</UI> → <UI>ログトレンド</UI>
    -   <UI>検索機</UI> → <UI>ログ検索</UI>

-   <Status>Change</Status> <UI>ログ検索</UI> メニューで照会内容を <UI>Timestamp</UI> を基準にソートする機能（<UI>新しい順</UI>/<UI>古い順</UI>）に変更

-   <Status>Feature</Status> <UI>ログトレンド</UI>から検索キーを最大2つまで選択できるように修正（検索値は制限なし）

-   <Status>Change</Status> <code>Server</code> プロジェクトの <UI>ログ設定</UI> メニューでエージェント設定オプションを変更

#### 警告通知

-   <Status>Feature</Status> <UI>イベント設定</UI> メニューで <UI>エージェント重複通知</UI> オプションを追加

    > モニタリング中のエージェントのうち、同じIDを設定したエージェントが2つ以上ある場合、メトリクスを重複して収集することによって発生する警告通知です。 エージェント名が重複して設定されていないかエージェント設定で関連オプションを確認してください。ブルー/グリーンのようなデプロイ状況でエージェントが重複して警告通知が発生した場合、この機能を利用することをお勧めします。 同名のエージェントが同時に2つ以上起動した場合、選択した時間中は通知が発生しません。

-   <Status>Feature</Status> <UI>イベント受信設定</UI> メニューで <UI>受信タグの未設定通知を受け取る</UI> オプションを追加

    > **_イベント受信タグ_**が設定されていない警告通知を受信しない場合は、そのオプションを選択してください。
