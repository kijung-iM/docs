**DB情報入力**段階を実行する中で接続に失敗した場合は、次の内容を参考にして問題を解決してください。

## 接続失敗後のリソース整理

DB情報を誤って入力したりファイアウォールが開放していない場合は、次のようなメッセージが表示されます。

![DB接続失敗](/img/mysql-rds-db-info-fail.png)

1.  **_失敗したエージェントの削除_**ボタンをクリックしてエージェントリソースを整理してください。

2.  「**_作成に失敗したエージェントを削除中です_**」というメッセージとともに削除作業を進めます。

3.  削除が完了したら、**DB情報入力**画面に戻ります。

接続の失敗原因によるメッセージを参考にして、原因を解決してください。

## 接続失敗事例

### IPアドレスまたはポート番号エラー

![IPまたはポート番号エラー](/img/mysql-rds-db-fail-01.png)

-   IPアドレスまたはポート番号を正しく入力したか確認してください。
-   モニタリング対象DBのファイアウォールを開放し、外部接続を許可したか確認してください。

### モニタリング用アカウント入力エラー

![アカウント入力エラー](/img/mysql-rds-db-fail-02.png)

-   モニタリング用に作成したアカウントのIDとパスワードを正しく入力したか確認してください。
-   モニタリング用アカウントに必要なすべての権限を付与したか確認してください。

:::note

モニタリングアカウント作成の詳細については、[次の文書](install-agent#uid)を参考にしてください。

:::
