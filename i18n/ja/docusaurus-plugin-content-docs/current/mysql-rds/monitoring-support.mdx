モニタリングを開始する前に、データベース対応バージョンとエージェントアーキテクチャ、インストール要件などを確認してください。 

WaTab MySQL RDSモニタリングサービスは、データベース接続に成功するとすぐにモニタリングを開始できます。 その製品を選択し、プロジェクトを作成してください。DB接続情報を入力すると、DBエージェントがWhaTapサーバーに生成され、必要な設定を自動的に実行します。設定を完了すると、エージェントが自動起動してモニタリングデータを収集する仕組みです。そのため、WhaTap収集サーバーで顧客DBに接続することを許可する必要があります。収集データの転送は、セキュリティのためにSSL通信を使用します。RDSは、Public Subnetに位置し、WhaTap収集サーバーへのアクセスを許可する必要があります。

## データベース対応バージョン

-   **MySQL RDS**製品は、DBXエージェントとDB区間のSSL通信を保障する**AWS RDS DB**での使用をお勧めします。
-   **AWS RDS MySQL**のべてのバージョン（5.7~8.0）、**AWS RDS MariaDB**のすべてのバージョン（10.3~10.6）に対応します。

## アーキテクチャ構成

既存の[MySQLモニタリング](https://docs.whatap.io/mysql/monitoring-intro)製品は、収集エージェントを顧客ネットワーク内にインストールする必要があります。一方、**MySQL RDS**は、DBエージェント(DBX)がWaTabサーバーに位置します。DB接続さえ可能な環境であれば、エージェントを自動インストールしてモニタリングを開始します。 このため、ファイアウォールの設定が必須です。

![MySQL RDS](/img/mysql-rds-architecture.jpeg)

## ファイアウォール

DBサーバーとWhaTap収集サーバーを接続するためには、DBサーバーでファイアウォールを開放し、外部接続やDBへのアクセスを許可する必要があります。したがって、モニタリング対象の**AWS RDS DB**は、**Public subnet位置**でなければならず、Security Groupで**WhaTapサーバーへの接続を許可**しなければなりません。

![방화벽](/img/mysql-rds-firewall.png)

-   ソース：**54.180.11.176**
-   ターゲット：モニタリング対象DBのポート番号

:::note

以下の事項に該当する場合は、既存の[MySQLモニタリング](https://docs.whatap.io/mysql/monitoring-intro) 商品を利用してください。

-   RDSがprivate subnetにある場合
-   DBに外部接続を許可できない場合

:::

## セキュリティー

WhaTapは、顧客のデータを保護するために2つの側面のセキュリティを強化しました。収集データの転送は、SSL通信を利用します。 DB接続情報は3rd-partyソリューション（Vault）を適用し、WhaTapサーバーに暗号化して保存します。
