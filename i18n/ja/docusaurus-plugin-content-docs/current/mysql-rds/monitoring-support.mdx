モニタリングを開始する前に、データベース対応バージョンとエージェントアーキテクチャ、インストール要件などについて確認してください。 

WaTab MySQL RDSモニタリングサービスは、データベース接続に成功するとすぐにモニタリングを開始します。 その製品を選択し、プロジェクトを作成してください。DB接続情報を入力すると、DBエージェントがWhaTapサーバーに作成され、必要な設定が自動的に実行されます。設定が完了すると、エージェントが自動で起動し、モニタリングデータを収集します。そのため、WhaTap収集サーバーで顧客DBに接続できるように許可する必要があります。収集データの転送は、セキュリティのためにSSL通信を使用します。RDSは、Public Subnetに配置され、WhaTap収集サーバーへのアクセスを許可する必要があります。

## データベース対応バージョン

-   **MySQL RDS**製品は、DBXエージェントとDBセグメント間のSSL通信を保証する**AWS RDS DB**での使用を推奨します。
-   **AWS RDS MySQL**のべてのバージョン（5.7~8.0）、**AWS RDS MariaDB**のすべてのバージョン（10.3~10.6）に対応しています。

## アーキテクチャ構成

既存の[MySQLモニタリング](https://docs.whatap.io/mysql/monitoring-intro)製品では、収集エージェントを顧客ネットワークにインストールする必要があります。一方、**MySQL RDS**では、DBエージェント(DBX)は、WhaTapサーバー上にあります。DB接続のみ可能な場合は、エージェントが自動的にインストールされ、モニタリングを開始されます。 これらの目的のために、ファイアウォールの設定が必要です。

![MySQL RDS](/img/mysql-rds-architecture.jpeg)

## ファイアウォール

DBサーバーとWhaTap収集サーバーを接続するには、DBサーバーでファイアウォールを開放し、外部接続とDBへのアクセスを許可する必要があります。しかしながら、モニタリング対象の**AWS RDS DB**は、**Public subnetの場所**にあるため、Security Groupで**WhaTapサーバーへの接続を許可**する必要があります。

![ファイアウォール](/img/mysql-rds-firewall.png)

-   ソース：**54.180.11.176**
-   ターゲット：モニタリング対象DBのポート番号

:::note

以下のいずれかに該当する場合は、既存の[MySQLモニタリング](https://docs.whatap.io/mysql/monitoring-intro) 製品を利用ください。

-   RDSがprivate subnetにある場合
-   DBへの外部接続を許可できない場合

:::

## セキュリティー

WhaTapは、顧客のデータを保護するために2つの側面のセキュリティを強化しました。収集データの送信はSSL通信を利用します。 DB接続情報は3rd-partyソリューション（Vault）を適用し、WhaTapサーバーに暗号化して保存します。
