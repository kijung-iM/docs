**ブラウザーモニタリング(Browser Monitoring)**は、ブラウザーアプリケーションのページ読み込み時間、サーバーレスポンス時間、リソース読み込み時間、ブラウザーエラーなど、様々な性能指標を収集・分析します。 収集・分析したデータにより、ブラウザーアプリケーションのボトルネックや性能問題を把握し、最適化できます。 また、ブラウザーごとに性能の違いを把握することができ、最高のユーザーエクスペリエンスを提供できるデータを提供します。

## **ブラウザーモニタリング**、なぜ必要なのか？

-   **クラウドとMSA時代の複雑なサーバー環境におけるブラウザーアプリケーションのレスポンス時間の最適化**

    クラウド、MSAなどの登場に伴い、サーバー環境がさらに複雑になりました。 このように複雑になった環境では、個々のサーバーの性能指標だけではブラウザーアプリケーションのレスポンス時間が遅れる原因を把握することは困難です。 それで、サーバーとクライアント間のネットワーク状況やブラウザーの読み込み時間を含む要素を考慮する必要があります。 また、レスポンス時間に影響を与えるブラウザーのリクエストデータのサイズやレスポンスサイクルなどの関連要素も考慮する必要があります。

    実ユーザーのレスポンス時間およびユーザーエクスペリエンスを最適化するためには、**ブラウザーモニタリング**が必要です。

-   **実ユーザー環境で発生する予測不可能なエラーに対する迅速な対応**

    開発、修正、配布が頻繁に発生するCI/CDが導入されることにより、実際のユーザーに発生するエラーは配布前のテストだけでは予防できません。 実ユーザーがウェブページを利用して発生するエラーについては、ユーザーのデバイスやブラウザーの種類、ネットワーク状態などによって様々なエラーが発生することがあります。 

    **ブラウザーモニタリング**を通じて実際のユーザーに発生するエラーをモニタリングし、これを予防または最小限に抑える技術的な方法を適用する必要があります。

-   **様々なブラウザーとデバイスで最適化されたユーザーエクスペリエンスを提供するための統計データの提供**

    ブラウザーの種類とデバイスが多様になるにつれて、実際のユーザーが経験するエラーをモニタリングする必要があります。 例えば、特定のブラウザーまたはデバイスに限って発生するエラーを確認できます。 これにより、ユーザーエクスペリエンスを改善し、より良いサービスを提供できます。

-   **SPA環境におけるブラウザーアプリケーションの性能分析と最適化**

    ウェブアプリケーションであるSPA(Single Page Application)では、主にブラウザーでウェブページを実装します。 以前は、ウェブページのすべての要素がサーバーでレンダリングされていましたが、今ではブラウザーでページをレンダリングし、必要なデータだけをサーバーから読み込んで使用します。 このようにウェブアプリケーションが複雑になり、サーバーの管理だけで性能が速いか遅いかを判断することはできません。 したがって、**ブラウザーモニタリング**を通じてブラウザーアプリケーションの問題を見つけ、最適化を実行する必要があります。

-   **1st Partyと3rd Partyリソースの総合的なレスポンス時間測定及び最適化によるユーザー体感性能の改善 **

    ウェブサービスの実ユーザーの体感性能は、社内で直接運用する1st partyと外部で運用する3rd partyのレスポンス時間を総合的に測定しなければなりません。 3rd partyリソースのレスポンス時間が遅れるだけで、全体レスポンス時間が遅れる可能性があります。

## 主な機能

**ブラウザモニタリング**は、エンドユーザーの立場からウェブサイト利用の問題有無、エンドユーザーの使用性、エンドユーザーの環境をモニタリングします。 次の主な機能を提供します。

### ページ読み込み性能情報提供

![Page load timeline](/img/browser-pageload-timeline.png)

![Page load details](/img/browser-pageload-details.png)

ブラウザがウェブページをどれだけ迅速に読み込むかをモニタリングします。 ユーザーは迅速に読み込まれるWebページを好むため、ページ読み込み性能情報を通じて開発者はWebページの性能を最適化できます。 ページ読み込み性能情報は、ブラウザーがページを読み込むのにかかった時間、ページ要素の読み込み時間、サーバーレスポンス時間などを提供します。 この情報により、ウェブページの性能を改善する方法を探し出せます。

### AJAX性能情報提供

![AJAX hitmap](/img/browser-ajax-hitmap.png)

AJAXを通じてサーバーとデータをやり取りする時の性能情報をモニタリングします。 AJAXは、ウェブページがサーバーと非同期的にデータを交換し、ページを更新しなくてもブラウザーアプリケーションをアップデートできるようにする技術です。 **ブラウザーモニタリング**は、AJAXリクエストとレスポンス時間の測定、サーバーレスポンス時間の分析、ネットワーク待機時間など多様なデータを収集します。 収集データは、ブラウザーアプリケーションの性能向上とユーザーエクスペリエンスの改善に役立ちます。

### ウェブページリソース情報の提供

![Web page resource list](/img/browser-page-resource-list.png)

ブラウザーがサーバーからダウンロードするリソースは、ページの読み込み性能に大きな影響を与えます。 リソース一覧では、読み込み速度が遅いか、ファイルサイズが大きいリソースを迅速に把握することができます。 これらの情報を使用して、ウェブページの性能を向上させ、ユーザーエクスペリエンスを改善するために活用できます。

### ブラウザーラー情報提供

![Browser error information](/img/browser-error-info.png)

ユーザーがウェブサイトを利用する過程で発生するブラウザーエラー情報を収集します。 この情報により、開発者はユーザーがウェブサイトを利用しながら経験する問題を把握し、迅速に対応できます。 ユーザーエクスペリエンスを改善し、ウェブサイトの安定性を高めることができます。

### コアウェブバイタル情報提供

![Core web vitals](/img/rum-core-web-vitals.png)

コアウェブバイタルは、グーグルで定義したウェブページの性能に対する重要な指標で、ページ読み込み時間とレスポンス時間、レンダリング時間、ユーザー相互作用時間などがあります。 これらの情報を収集して分析することで、ウェブサイトを上位に露出するのに役立ちます。

### ユーザー接続環境分析

![User environments](/img/browser-user-env.png)

ユーザーのデバイス情報やブラウザー情報、オペレーティング システム情報、地域情報など、様々なユーザーの接続環境を分析します。

特定のデバイスで問題が発生した場合は、そのデバイスの情報を分析して問題を解決できます。 また、特定の地域でウェブアプリケーションが遅く動作する場合は、その地域の情報を把握して最適化を行うことができます。 ユーザーが使用するブラウザーやオペレーティングシステムなどを把握し、ウェブアプリケーションの互換性問題を解決するために必要な情報を提供できます。

## **ブラウザーモニタリング**を開始する

次の手順に従って**ブラウザーモニタリング**を開始できます。

1.  WhaTapのアカウントがない場合は [会員登録](https://www.whatap.io/ko/signup/)を進めてください。

2.  ブラウザーの互換性と事前設定をチェックしてください。

3.  [WhaTapモニタリングサービス](https://service.whatap.io)に移動し、ブラウザープロジェクトを作成してください。

4.  ブラウザーエージェントを適用してください。
