この文書では、WhaTapモニタリングサービスの一つであるサーバモニタリングの**_リソースボード_**メニューのチャート型ウィジェットに対する分析方法を説明します。サーバー（Server）の問題を**_リソースボード_**でどのように特定して分析する方法を説明します。**_リソースボード_**の機能の概要については、[次の文書](https://docs.whatap.io/server/resourceboard)を参照ください。

サーバーモニタリングの中心はプロセスです。プロセスが通常の動作範囲内で中断することなく処理を動作させるためには、モニタリングを通じてサーバーの状態をいち早く把握し、さまざまな障害に対応できる必要があります。リソース使用量の検索による推測がその始まりです。WhaTapの**_リソースボード_**は、データ集約型の画面でリソース使用量を一目で確認できるようにCPU、Memory、Disk、Network関連指標で構成される時系列チャートウィジェットが用意されます。

-   サーバー状態の概要：**_Server_**、 **_OS_**、 **_Total Cores_**、 **_Avg CPU_**、 **_Avg Memory_**、 **_Avg Disk_**、 **_Server Status Map_**
-   システム動作統計：**_CPUResourceMap_**
-   早期警報及び通知：**CPU TOP5**、**Memory TOP5**、**Disk I/O TOP5**、**プロセスCPU TOP5**、**リアルタイムアラート**

## サーバモニタリングの主要なメトリック

-   **CPU**：CPUのパフォーマンス指標は、サーバのパフォーマンスを把握するための最も重要な指標として使用します。CPUの使用率が一定量以上を超えると、サービスに影響が始まります。問題が発生した場合、ハードウェアの追加購入やCPUを使用するアプリケーションのパフォーマンスをチューニングするなど、CPU使用率の管理目標を達成するための手順を実行することができます。

    :::note

    IT関連のソリューションは、サーバーのパフォーマンスに基づいて価格設定があります。それらのほとんどが、CPUのコアに基づいて価格が設定されているのと同じ理由です。 

    :::

-   **Memory**：バッファおよびキャッシュメモリを含め、メモリの使用状況を確認します。メモリの使用量が常に減少している場合や持続的に使用量が減少しない場合は、メモリ使用量の管理目標を達成するための手順を実行することができます。

-   **Disc I/O**：DiskI/Oはネットワークドライブを使用する場合、必ず確認しなければならないモニタリング要素です。ディスクの読み取り速度、書き込み速度、キュー、待ち時間の割合などをモニタリングします。

-   **Network**：ネットワーク指標は、ネットワークインターフェースの入出力トラフィックの速度とエラーパケットなどをモニタリングします。

## サーバステータスの概要

![インフォパネル](/img/resourceboard-info.png)

ユーザーは**_リソースボード_**の上部情報パネルからプロジェクトに登録されているサーバー全体の概要指標を簡単に確認することができます。 

**_Server_**ウィジェットでサーバー合計数と障害が発生したサーバーの数を表示することができます。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_サーバーの一覧_**メニューに移動します。**_状態_**カラムでアクションが必要な場合があります。![危険アイコン](/img/ic-warning-critical.svg)<span class='vslow'>危険</span>(<span class='vslow'>赤色</span>)、今後のトラブルの可能性をお知らせ![警告アイコン](/img/ic-warning-org.svg)<span class='slow'>警告</span>(<span class='slow'>橙色</span>例外ステータスと![正常アイコン](/img/ic-check-circle-filled.svg)<span class='snormal'>普通</span>(<span class='snormal'>緑色</span>)、![非アクティブアイコン](/img/ico-disable-agent.svg)<span class='k8s-normal'>非アクティブ</span>(<span class='k8s-normal'>グレー</span>)の状態は、アイコンと色で簡単に区分して確認することができます。**_サーバー一覧_**の詳しい説明は[次の文書](https://docs.whatap.io/server/server-list)を参照ください。

***OS***ウィジェットはプロジェクト内のOS数、***Total Cores***はサーバー全体コアの合計、***Avg CPU***はサーバー全体CPUの平均使用量、***Avg Memory***はサーバー全体メモリの平均使用量を表します。

また、**_リソースボード_**中央メインチャートの![サーバーのステータスマップアイコン](/img/dashboard-honeycomb.svg)アイコンを選択し、**_ServerStatusMap_**に移動すると、ハニカムチャートからプロジェクト内サーバーのステータスを一目で確認できます。ハニカムチャートビューは、サーバーの台数が多い場合に便利です。1つの六角形が1つのサーバーエージェントを意味します。問題が発生したサーバーを色で視覚化し、直感的に識別が可能です。個別の六角形を選択すると、そのサーバーエージェントの**_サーバー詳細_**ページに移動します。

![六角形チャートcht](/img/honeycomb-view.png)

WhaTapの**_リソースボード_**の各ウィジェットを活用して、リソース使用量全体とサーバーの状態の概要を簡単に調べることができます。 

## システム運用の統計

プロセスの正常動作を確認するために最も重要な要素はCPUの使用量です。CPU指標がシステムの負荷状況を最も早く反映するためです。WhaTapはシステム全体の運用状況を一目で確認できるようCPU使用量を分布図チャートとして提供します。**_リソースボード_**中央の![リソースマップアイコン](/img/dashboard-hitmap.svg)アイコンを選択すると、**_CPUResourceMap_**ウィジェットの参照が可能です。**_CPUResourceMap_**ウィジェットでプロジェクト内のサーバー全体のCPU使用量分布図を確認することができます。セル領域をドラッグして、セクションの詳細情報を表示します。

![リソースマップのリソースボード](/img/resourceboard-resourcemap.png)

クラウドIT資産の効率的に運用するには、**リソースの過剰条件**と**過不足**の状態をすべて**回避し**、**使用量**を**適切に制御**することが必要です。次のサンプル画面は、操作を最適化のためにリソース使用量基準を50%前後とし、コストおよびパフォーマンス効率のバランスを追求します。例のような場合、使用量が増加した時に50%のパフォーマンスマージンで一時的な障害を回避できます。

![40%以下のリソースマップ](/img/resourcemap-under40.png)

過剰投資とは、サービスと比較してシステムが過剰にインストールされ、インフラコストが必要以上に支出されている状態です。システムの安定性が保証されているため、ユーザーがピーク時に受ける影響は大きくありません。そのため、モニタリングに投資する時間を減らすために、使用量の平均値を下げるために過剰投入する傾向があります。ただし、過剰投入の傾向が高まるとコストが増加するため、状況をすばやく特定することが重要です。過少投入とは、システム変更などによる負荷増加によりシステム使用量が管理範囲を超えた場合に発生し、サービス品質の低下やユーザビリティの低下を引き起こします。ユーザーがサービスから離れる前に、適切なアクションが必要です。

WhaTapの**_CPU Resource Map_**は、過剰投入と過少投入の状況を簡単に特定して確認することができます。マップの下段は一般的に過剰投入傾向にあり、上段は過少投入傾向にあります。 

![リソースマップの例cht](/img/temp-resourcemap.png)

サンプル画面**_CPUResourceMap_**下部にある![number-s](/img/number-01.png)エリアをドラッグして詳細情報を表示すると、**_サーバー当たりCPU使用率_**グラフで次のように過剰投入の傾向が確認できます。

![過剰投入cht](/img/cpu-usage.png)

これは高い傾向性を提示したもので、下段で表示されるサーバーがすべて過剰投入の状態ではありません。定期的に負荷がピークになるサーバーの場合、チャートの解析に注意する必要があります。

## 早期アラートおよび通知

WhaTapの**_リソースボード_**は、リソース使用量が最も高いサーバーとプロセスの上位5つの一覧チャートを提供します。ユーザーが問題のあるサーバーを特定するのに役立つ早期警報ウィジェットで、それぞれ右側と下段で確認できます。

リソース使用率が高い場合、問題が必ず発生します。WhaTapは、Top5の一覧チャートを通じてサーバーモニタリングの重要な指標であるCPU、Memory、Disk I/O、Networkなどのリソース使用率が高い重要な対象サーバーとCPU、Memory使用率が高いプロセスを一目で確認し、問題を早期に解決するのに役立ちます。 

これは、より大きな問題が発生する前に、WhaTapの**_リソースボード_**を通じてシステムの主要な問題の事例を紹介します。次の画面では、**_Disk I/O Top5_**ウィジェットに、使用可能な容量に対してDisk I/Oが高い機器のステータスが表示されます。 

ステータス情報によって同一機器で発生した障害履歴を確認し、速度低下やインデックスファイルの破損などの異常が発生していることを確認しました。そのため、SSDや大容量の導入が難しい場合、NASスナップショットのバックアップを利用することで、Disk I/Oの問題を解決しました。

![diskスペックの変更cht](/img/discio-usage.png)

最近のイベントの傾向を確認できる**_リアルタイム通知_**は**_リソースボード_**の上部にある固定メニューで利用できます![リアルタイム通知icon](/img/ico-noti.svg)ボタンをクリックして確認することができます。WhaTapは、独自で通知するしきい値設定を提供します。エージェントをインストールしてデータが収集されると、プリセットなしで、それ以降の通知を確認できます。例えば、Disk I/O(%)指標は5秒間のディスク使用率を表します。Disk I/O(%)が80%を超えるとシステムパフォーマンスに影響を与える可能性がありますが、100%ならディスクが常時稼働していることを表します。WhaTapサーバーモニタリングでデフォルトに設定されたDisk I/O(%)のアラート値は90%です。つまり、90%を超える場合は、ユーザーがしきい値の詳細設定していない場合でも、通知がすぐに行われます。

![リアルタイム通知cht2](/img/server-rt-notice.png)

WhaTapのデフォルトの通知設定により、ユーザーは、エージェントのインストールされるとすぐに、モニタリングが開始されます。WhaTapサーバーモニタリングの特長である効率と利便性を反映します。簡単に設定できるプロセス通知など、サーバーモニタリング通知設定の詳細については、[次の文書](https://docs.whatap.io/server/warning-notice))を参照ください。

**_リソースボード_**はサーバー全体の現況情報を要約して簡潔でイシュー中心に見られる効率的なダッシュボードです。CPUイシューが存在するサーバーの台数と推移を同時に感知するためのメインチャート(**_CPU Resource Map_**)とOSモニタリングの核心指標別Top5の一覧を通じてイシュー発生可能性が高い対象資源を露出させ、サーバーで発生した通知履歴を最新順に表示します。WhaTapの直感的で簡潔なダッシュボード構成は、大規模なシステムをモニタリングする必要がある場合により便利です。

:::note 

**ダッシュボードの追加活用**

-   リソース別Top5ウィジェットの![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_リソースイコライザー_**メニューへ移動し、サーバー全体のリアルタイムの使用状況を見ることができます。 
-   サーバーユニットの詳細情報は、ウィジェット内のチャート領域を選択して移動する**_サーバー詳細_**メニューで確認できます。 
-   サーバーリソースの消費パターンを表示したり、負荷設計と比較する場合は、**_分析_**>**_メトリックチャート_**メニューを使用します。

:::
