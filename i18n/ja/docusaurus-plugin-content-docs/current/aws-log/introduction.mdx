AWS上で発生するリソースログは、CloudWatch Logを通じて収集されるログの行単位の文字列情報と、S3にアーカイブ形で保存されるログの形に区分されます。 一般的に、ユーザーは次のようにS3とLog Groupにそれぞれ属して、リソースごとに確認する煩雑なプロセスを実行する必要があります。 

![none aws-log sc](/img/aws-log-none.png)

WhaTap AWS Logは、AWS上で発生する各種ログを収集し、統合モニタリングするための機能を提供します。 AWS Logを活用する場合、ユーザーは上記の煩わしい過程は必要がありません。 AWSのリソースログが蓄積されるS3とCloudWatch Log GroupをWhaTap Forwarderがサブスクリプションし、ログ作成時にWhaTap環境に送信します。 

![aws-log sc](/img/aws-log-intro-c.png)

### 主な特長

-   WhaTap AWS Logを使用してリソースログを一度に収集できます。
-   WhaTap AWS Logを活用し、時間帯別ログ発生量を計算することができます。
-   WhaTap AWS Logを使用してログの内容を検索することができます。
