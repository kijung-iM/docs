## プロセスグループとは？

プロセスグループは、インスタンスに対して同じアクションを実行する複数のプロセスのグループです。個別のプロセスではなく、グループ単位でモニタリングまたは通知を受け取る場合は、プロセスグループを使用してください。

既存の各プロセスのリソース使用量の合計を一目で確認できます。個々のプロセスのリソース使用量を計算する必要がなくなります。

プロセスグループのリソース使用量の合計が特定のしきい値を超えたときに通知されるように指定できます。
通知の設定に関する詳細は、 [次の文書](warning-notice#process-event)を参考にしてください。

## プロセスグループの追加

<!-- Adminアカウントでログインした後、次の手順を進めてください。-->

:::note

プロジェクトの **修正権限**が必要です。 権限の詳細については [次の文書](../project/project-structure)を参考にしてください。

:::

1.  ホーム画面 > プロジェクト選択 > **_サーバー一覧_** > **_サーバー詳細_**に移動し、プロセスリストの **_グループ_** ボタンをクリックします。<br/>
    ![サーバー詳細プロセス一覧](/img/server-detail-process.png)
2.  プロセスグループ設定ウィンドウが表示される場合、グループに含めるプロセス名パターンを**_パターン_**に、前後のアスタリスク(\*)を含めて入力します。

    > 例） _whatap／_

    ![](https://img.whatap.io/media/images/process_group_details.png)

    入力後、右側の ![虫眼鏡アイコン](/img/ic-search.svg)**_虫眼鏡_** ボタンを選択してください。 そのパターンに該当するプロセスの一覧を照会することができます。 プロセス名は、画面上に表示されるプロセス名を基準とします。
3.  **_Pattern 別名_**にプロセスグループ名を入力し、右側の **_適用_** ボタンを選択してください。 グループ化は**設定したプロジェクト**の **すべてのサーバー**で実行されます。

## プロセスグループの例

-   **プロセスグループの適用**\
      次の画面は **_パターン_**に \*tomcat\*を入力し、**_パターン別名_**をtest_groupとして適用した例です。
      ![](https://img.whatap.io/media/images/process_group_example.png)

-   **プロセスグループ化以前のプロセス一覧**\
      ![グループ化以前のプロセス一覧](/img/server-process-list.png)
      **_パターン_**用にグループ化するプロセスの一覧です。
    > -   例）javatomcat_left/bin/test.jarcom.virtual.App 
    > -   例）javatomcat_right/bin/test.jarcom.virtual.App

-   **プロセスグループ化の登録確認**\
      新しいプロセスグループtest_groupが次のようにプロセス一覧に登録されていることを確認できます。
      ![](https://img.whatap.io/media/images/test_group.png)

:::note

-   グループ化の適用後、プロセスグループで各プロセスに該当するデータを合算した数値を確認できます。 
-   グループ化する前にデータを表示するには、プロセス別にデータを表示します。

:::
