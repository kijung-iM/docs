ホーム画面 > プロジェクト選択 > **_アプリケーション_** > **_ヒートマップ_**

コンテナ内部アプリケーションのヒートマップ情報を確認できます。ヒートマップは、トランザクション実行件数マップでレスポンス時間分布図を表示します。検索条件は時間選択、区分、アプリケーションで、区分条件は次のとおりです。

| 分類          | 説明                                              |
| ----------- | ----------------------------------------------- |
| エージェント      | コンテナ内部にインストールされているアプリケーションエージェント名               |
| エージェント種類    | エージェントがグループ化されたエージェントの種類（`OKIND`）               |
| エージェントサーバー  | エージェントが実行中のKubernetesノード名                       |

![ヒートマップトランザクション](https://img.whatap.io/media/images/whatap_kubernetes_application_hitmap_transaction.png)

ヒートマップの特定区間をドラッグすると、トレース情報を確認できます。トレースデータは、URLベースの検索が可能であり、アクティブスタックが存在するトレース、マルチトランザクションで行うトレース情報をフィルタリングして確認することもできます。
