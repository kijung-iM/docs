ホーム画面 > プロジェクト選択 > **_ダッシュボード_** > **_コンテナマップ_**

WhaTap Kubernetesのコンテナマップは、Kubernetesモニタリングの開始点です。ユーザー設定の表示によって、各オブジェクトは1つのコンテナ、または1つのPodを表示します。

<!-- ![쿠버네티스 컨테이너 맵](https://img.whatap.io/media/images/whatap_kubernetes_container_map_intro.png)-->

![コンテナマップ](/img/kube-container-map-01.png)

## ![](/img/number-01.png)コンテナまたはPod

各オブジェクトの形状とデザインは、コンテナまたはPodの特性と状態を示します。

-   色：コンテナまたはPodの状態を意味します。状態の詳細については、[次の文書](container-status)を参考にしてください。
-   枠線
    -   オブジェクトの中には、境界線が表示されます。これはコンテナに、Java、Node.js、Pythonなどのマイクロエージェントをインストールする対象です。 
    -   1つのPod内に2つのコンテナがあり、2つのうち1つのコンテナにマイクロエージェントをインストールした場合、境界線が表示されます。つまり、Pod内のコンテナのいずれか一つでもマイクロエージェントがインストールされている場合は、境界線が表示されます。

## ![](/img/number-02.png)クラスタープロジェクト - ネームスペースプロジェクトの移動

![コンテナマップのプロジェクト一覧](https://img.whatap.io/media/images/whatap_kubernetes_container_map_namespace_project_list.png)

クラスタープロジェクトの下にネームスペースプロジェクトを作成した場合、コンテナマップの画面上にあるプロジェクト一覧ボックスを使用してクラスタープロジェクトを含むプロジェクト一覧の確認およびプロジェクト間の移動が可能です。

:::note

-   WhaTap Kubernetesサービスの**クラスタープロジェクト**は、**マスタープロジェクト**の新しい名前です。
-   ネームスペースプロジェクトの作成に関する詳細は、[次の文書](namespace)を参照してください。
:::

## ![](/img/number-03.png)ユーザー設定

![ユーザー設定](/img/container-map-custom-view.png)

コンテナマップには、複数のユーザー設定が用意されており、目的の画面表示を作成する機能が画面順に用意されています。

-   カテゴリ：**_Container_**または**_Pod_**が選択できます。

-   **_グループ化_**(クラスタリング)：グループ化できる条件を選択できます。コンテナまたはPodを選択した条件に応じてカードで区切って表示します。詳細は[次の文書](container-group)を参照してください。

-   **_ConState(state)_**：チャート指標を選択できます。

-   ![設定アイコン](/img/ico-setting.svg)ボタンをクリックすると、各オブジェクトにコンテナまたはPodの情報を表示できるオプションを選択できます。

    ![オプション](/img/k8s-container-map-setting.png)

    -   **_WhaTapエージェントの表示_**：コンテナまたはPodの情報を表示できます。目的の**_ラベル_**オプションを選択します。

    -   **_カードの下部にメトリクスを表示する_**：画面下部にメトリクス情報を表示するかどうかを選択できます。

    -   **_1分以内に作成されたコンテナを表示する_**：作成されてから1分以内のコンテナの場合、明るい色で表示するオプションです。

-   ![フィルターアイコン](/img/ico-filter.svg) **_フィルター_**：ユーザーが選択した項目に基づいてフィルタリングし、コンテナまたはPodの状態を確認できます。

-   ![リセットアイコン](/img/ico-reset.svg) **_リセット_**：ユーザーが設定した画面表示を初期化します。

## 概要表示と詳細表示

ユーザーは、関心のあるコンテナまたはPodを選択して概要および詳細情報を確認できます。この情報には、次のような主要なモニタリングデータが含まれます。

-   トレース（Trace）
-   メトリクス（Metrics）
-   ログ（Log）
-   イベント（Event）

これらのデータに基づいて、ユーザーはKubernetes環境のObservabilityを得ることができます。以下の**概要を見る**、**詳細を見る**画面は、いずれも上記の4項目を中心とした情報を提供します。

### 概要を表示

特定のオブジェクトまたはオブジェクトのセットをクリックします。右側に表示される**概要を見る**画面を通じて該当オブジェクトの概要を確認することができます。

![概要を見る](https://img.whatap.io/media/images/whatap_kubernetes_container_map_right_summary.png)

### 詳細を見る

**概要を見る**画面右上の![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンをクリックします。**詳細を見る**画面に切り替えます。

![詳細を見る](/img/container-map-detail-mode.png)

## リアルタイム及び履歴照会

コンテナマップは、基本的にリアルタイム参照ベースですが、複数の時点に基づいて参照機能を提供します。リアルタイムの変更は、画面右上の時間セレクタを使用します。

![時間セレクタ](/img/kubernetes-time-seletor.png)

| 追跡ステータス        | 説明                                         |
| -------------- | ------------------------------------------ |
| リアルタイム参照（LIVE） | 基本モード<br/>10秒間隔でリアルタイムステータス変更              |
| 一時停止の参照（PAUSE） | リアルタイムステータス変更OFF<br/>現在のステータスを詳しく確認する時に使用  |
| 過去の時点参照        | ユーザーは、過去の特定の時点を指定して確認<br/>年/月/日/時/分を指定できます |
