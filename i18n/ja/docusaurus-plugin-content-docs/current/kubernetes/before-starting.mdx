## クラスターおよびプロジェクトの構成

WhaTap Kubernetesモニタリングサービスは、1つのユーザーKubernetesクラスター環境を対象に1つ以上のWhaTapプロジェクトとして管理します。

ユーザーのKubernetesクラスターは、WhaTapサービス上で基本的には1つの**クラスタープロジェクト**として表現します。初めてKubernetes製品のプロジェクトを作成する場合、該当プロジェクトが**クラスタープロジェクト**になります。もしKubernetesクラスター内に複数のネームスペースが存在し、コンテナ/ポッド(Pod) の数量が多くなってプロジェクトを分けるには、Kubernetes内のネームスペース別に**ネームスペースプロジェクト**を作成して分離することができます。

![Kubernetesプロジェクト](/img/k8s-kuber-namespace.png)

### ![](/img/number-01.png)クラスタープロジェクト

-   Kubernetesクラスター内のすべてのリソース情報をモニタリングするためのプロジェクトです。

-   **クラスタープロジェクト**を通じてマスターエージェントとノードエージェントのインストールを進めます。

-   プロジェクト一覧で**クラスタープロジェクト**は、プロジェクト名の横に`CP`タグを、一番下に`Kubernetes`タグを表示します。

-   **クラスタープロジェクト**の作成の詳細については、[次の文書](install#create-project)を参考にしてください。

### ![](/img/number-02.png)ネームスペースプロジェクト

-   Kubernetesクラスターの下位ネームスペース（Namespace）単位のプロジェクトです。

-   ネームスペース（Namespace）単位のリソースモニタリングおよびアプリケーションコンテナのモニタリングを行うためのプロジェクトです。

-   プロジェクト一覧で、**ネームスペースプロジェクト**は、一番下に`Namespace`タグを表示します。

-   **ネームスペースプロジェクト**管理の詳細については、[次の文書](namespace)を参考にしてください。
