WhaTapモニタリングサービスを使用するためには、[会員登録](https://www.whatap.io/ko/signup/) 後にプロジェクトを作成し、そのURLを追加する必要があります。 会員登録に関する詳細については、[次の文書](../account/account-manage)を参考にしてください。

次の動画ガイドを参照してください。

{@youtube: uBa6AtXFlHk}

## プロジェクト作成

URLを追加する前に、まずプロジェクトを作成してください。

1.  [WhaTapモニタリングサービス](https://service.whatap.io)に移動し、ログインしてください。

2.  プロジェクトを作成するには、画面左側のメニューの **_+ <TR sid='project_add_btn'/>_** ボタンを選択してください。

3.  **_商品カタログ画面_**でURLモニタリングプロジェクトを作成してください。

    ![urlモニタリングプロジェクト](/img/url-project.png)  

4.  **_プロジェクト名_**、 **_タイムゾーン_** などの項目を設定してください。

5.  **_データサーバー地域_**でWEB-Checker-Regionを選択してください。

6.  すべての設定を完了した後は **_プロジェクト作成_** ボタンを選択してください。

7.  プロジェクト一覧で新規プロジェクトを確認してください。

:::note

-   **_データサーバー地域_**銀リージョン（クラウドサービスを提供するためにインストールしたデータセンターのグループ）を意味します。 特定のリージョンを選択すると、そのリージョンに属するデータセンターにユーザーのデータを保存します。
-   **_タイムゾーン_**は通知、レポートを生成する基準時間です。
-   複数のプロジェクトをグループ化して管理するには **_プロジェクトグループ_**からグループを選択するか、グループを追加してください。 グループの詳細については、[次の文書](../project/group)を参考にしてください。
:::

## URLを追加する

1.  生成されたURLプロジェクトを選択してください。 URLが設定されていない画面が表示されます。
2.  画面右上の **_+ URL追加する_** ボタンを選択してください。 
3.  追加された入力フィールドにURL設定を進めます。
    ![url入力フィールド画面](https://img.whatap.io/media/images/Screenshot_2021-01-15_ddd_15299_-\_URL_Monitoring.png)
    -   **_名前_**<br/>
          URL設定の固有識別情報として使用されます。 重複不可であり、リセット時に同じ名前で登録する場合は、同じ名前の統計データを同じURLのデータとしてみなします。
    -   **_URL_**<br/>
          チェック対象のURLを登録します。 呼び出し先URLのフルパスを入力します。http/httpsプロトコルに対応します。
    -   **_確認地域_**<br/>
          WhaTapが提供するエージェントを選択します。 Seoul、Tokyo、Singapore、Mumbai、Frankfurt、Califoniaリージョンを提供します。 次の確認地域選択の案内を参考にしてください。
4.  編集が完了したら **_貯蔵_** ボタンで設定情報を保存してください。 そして **_モニタリング_** ボタンを選択してモニタリング画面に切り替えてください。

## モニタリングする{#url-st}

<!-- 모니터링 화면으로 전환 후 모니터링을 수행합니다. -->

![URLモニタリング画面sc](/img/url-monitoring.png)

### 条件指定オプション

ホーム画面でプロジェクトを選択し **_URL一覧_** メニューに移動してください。 ![number-1](/img/number-01.png)領域上段のオプションを通じて表示形式や整列方式を指定したり、検索して一覧を照会することができます。

-   ![一覧ビューアイコン](/img/ic-list.svg)**_一覧ビュー_**：**_URL一覧_**を一覧形式で見ることができます。基本設定は **_一覧ビュー_**です。 
-   ![カードビューアイコン](/img/ic-project.svg)**_カードビュー_**：**_URL一覧_**をカード形式で見ることができます。
-   **_検索_**：入力ウィンドウを通じてURLまたは指定したURLの名前を基準に検索することができます。

![number-2](/img/number-02.png)領域で登録されたURL一覧の最終レスポンスステータスごとの件数情報を確認し、フィルタリングすることができます。 **_全体_** を見る、 **_9XX_**、 **_サーバーエラー_**、 **_リクエストエラー_**、 **_リダイレクション済み_**、 **_成功_** などを基準に照会することができます。 基本設定は **_全体_** を見るです。 

### URL一覧の書き出し、読み込み

![number-3](/img/number-03.png) 領域で現在 **_URL一覧_**をExcelファイル形式で書き込みまたは読み込みができます。 

-   ![書き出しアイコン](/img/ico-export.svg) **_書き出し_**：ウェブブラウザーから現在のURL一覧をExcelファイルとしてダウンロードします。
-   ![読み込みアイコン](/img/ico-import.svg) **_読み込み_**：Excelファイルを選択してURL一覧を読み込みます。 

<!-- ![URL 모니터링 화면](/img/url-added-feature.png) -->

### URLモニタリングの詳細

![number-4](/img/number-04.png) 領域のURL一覧のテーブルで、登録されたURL単位のモニタリングチェック結果を確認することができます。 各カラム項目は次のとおりです。 URLの登録・編集が必要な場合 ![編集アイコン](/img/ic-edit.svg) **_URL編集_** アイコンを選択し、編集画面に切り替えてください。

-   **_現在Status Code_**<br/>
      最終チェックレスポンスコード（Http Status Code）です。各ステータスコードを選択すると、HTTPステータスコードのまとめ文書に移動します。
-   **_名前/URL_**<br/>
      URL登録時に指定した固有の識別情報とURLです。
-   **_地域_**<br/>
      確認地域です。
-   **_経過時間_**<br/>
      20分間のチェック結果（レスポンスコードおよびレスポンス時間）または最終レスポンス時間です。 チャート領域からドラッグして時間範囲を照会する際、次のように該当時間帯のレスポンスコードおよびレスポンス時間を確認できます。
      ![詳細sc](/img/url-monitoring-dt.png)

:::note 

**WhaTap URL Monitoringサービスの使用**

URLモニタリングは、別途のポリシー変更があるまで無償サービスとして提供されます。 ユーザーは別のエージェントをインストールする必要はなく、WhaTapが提供するエージェントおよびサーバーリポジトリを活用します。

:::
