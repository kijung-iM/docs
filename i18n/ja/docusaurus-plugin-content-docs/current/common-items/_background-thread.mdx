## Background Threadに対するアクティブスタック

<!-- @since v1.6.2 -->

基本的にアクティブスタックは、トランザクションが実行されているスレッドのスタックです。 ただし、一部のバックグラウンドスレッドについてもスタックを分析する必要がある場合があります。 このとき、オプションにより、バックグラウンドスレッドのアクティブスタックを確保できます。 1.6.2バージョン以降からサポートします。

-   「async_stack_enabled」の値を「true」に設定するとアクティブ化されます。

    ```bash
    async_stack_enabled=false
    ```

-   スタックダンプの間隔は、フォアグラウンドアクティブスタックの設定に従います。

    ```bash
    active_stack_second=10
    ```

-   対象スレッド名を指定するときは、「\*」を使用して文字列パターンを指定します。 

    ```bash
    async_thread_match=http*,abc*
    ```

      スレッド名でスタックダンプ対象を識別します。「,」を使用してmatchを複数指定できます。

-   スタックのTopメソッドが「async_thread_parking」に登録されたクラス/メソッドの場合、スレッドがパーキング状態にあると判断してダンプを生成しません。

    ```bash
    async_thread_parking_class=sun.misc.Unsafe
    async_thread_parking_method=park
    ```
