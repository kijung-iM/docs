WhaTapデータベースモニタリングサービスは、Queryベースでパフォーマンス指標とユーザークエリを収集し、パフォーマンスモニタリングサービスを提供します。データベースのパフォーマンスをリアルタイムでモニタリングし、問題点を診断、分析できるソリューションを提供します。問題となるSQLを認識し、改善事項を適用した時に問題が解決されたかを確認することができます。

## データベースモニタリングの特長

-   **Public Cloud、 Private Cloudの両方に適用可能**

    クエリベースでデータを収集するため、モニタリング対象サーバーのOperating Systemに依存関係はありません。

-   **Agentless**

    モニタリング対象データベースサーバーにエージェントを直接インストールする必要はありません。RDS（AmazonRelationalDatabaseService）を含む任意のデータベースをモニタリングできます。

-   **様々なDBをリアルタイムに統合モニタリング**

    複数の異種DBの指標傾向を一目で可視化できるダッシュボードを提供します。１つのDBの詳細モニタリングダッシュボードを提供することで、パフォーマンス指標の傾向、実行中のSQL、ロック発生状況および通知発生履歴をリアルタイムで検出できます。

-   **事後分析**

    複数の指標トレンド、長期トレンド、複数の主要指標比較機能を提供することにより、特定状況の異常な状況を分析することができます。SQL、Lock Tree、Parallel Query統計、DB領域テーブルスペースなどの統計情報を確認することができます。特定のSQLのPlan情報を確認することで、SQLチューニングの手がかりが得られます。データベースエンジンパラメータの変更履歴をトレースできます。

-   **主要指標モニタリング**

    アクティブセッション、ロックツリー等、異常発生時に主要指標になり得る項目の詳細情報をリアルタイムでモニタリングできます。

-   **多数モニタリングの構造**

    複数のDBサーバーを管理する場合でも、統合された性能データ収集システムを通じて、大量にモニタリングデータの収集と統合ビューを提供します。

-   **パフォーマンスモニタリングデータ**

    DBベンダーが提供するパフォーマンス指標やリアルタイムにクエリデータを収集することで、問題分析やパフォーマンスに最適化で必要なデータを共有し、データに基づく意思決定を可能にします。

-   **クラウドDBモニタリング**

    AWS RDSなどのCloud DBのモニタリングができ、パフォーマンス指標やリソース統合モニタリングが可能です。

-   **統合モニタリングの構造**

    アプリケーション、サーバー、DBモニタリングを統一されたシステムで提供し、サービスの異常を特定する際、リンクされたビューを通じて原因を分析することが可能です。

-   **簡単な構造**

    DBサーバーにインストールすることなく、すぐに適用可能な構造で簡単にインストールできます。

-   **WhaTap SaaS**

    簡単なインストールプロセスで、データベースを即時、モニタリングできます。インストール型としても可能です。
