-   **profile_step_normal_count** <span class='type'>Int</span>

    デフォルト値 `1000`

    トランザクショントレースの基本ステップ数を指定します。

-   **profile_step_heavy_count** <span class='type'>Int</span>

    デフォルト値`1020`

    トレースの基本ステップを超えた場合、最大heavyステップ数を制限します。`profile_step_normal_count`で`profile_step_heavy_time`を超えるステップをカウントします。レスポンス時間のないステップは収集されません。

-   **profile_step_max_count** <span class='type'>Int</span>

    デフォルト値 `1024`

    トレースステップの最大数を設定します。収集されたトレースステップの数が設定値を超えると、その後収集されるステップはすべて捨てられます。`profile_step_heavy_count`を最大1000として設定すると、`profile_step_max_count`ほどのアクティブスタックが収集されます。

-   **profile_step_heavy_time** <span class='type'>MiliSecond</span>

    デフォルト値`100`

    Heavyステップの基準を指定します。指定された値より実行時間が長い場合は、`profile_step_normal_count`を超える場合でも`profile_step_heavy_count`以内で記録されます。

-   **profile_basetime** <span class='type'>MiliSecond</span>

    デフォルト値`0`

    トランザクションが設定値以下の時間内に終了した場合、トレース情報を収集しません。ただ、5分単位で最初に呼び出されたURL、エラーが発生したトランザクションに関するトレース情報は収集されます。

-   **query_string_enabled** <span class='type'>Boolean</span>

    デフォルト値`false`

    トランザクションURLのクエリストリングを一緒に収集する機能を活性化します。

-   **query_string_urls** <span class='type'>String</span>

    トランザクションからクエリストリングを収集するURLを登録します。複数登録するときはコンマ(_,_)を使います。  

-   **hook_method_patterns** <span class='type'>String</span>

    特定メソッドの応答時間を測定したいときに使用します。`[MODULE_NAME]:[CLASS_NAME].[FUNCTION_NAME]`または`[MODULE_NAME]:[FUNCTION_NAME]`形式で値を使用します。コンマ(,)区切り文字を通じて複数のメソッドパターンを指定することができ、モジュールに対する全クラスやクラスに対する全モジュールは、asterisk(\*)を使用して追跡できます。 

    :::note

    モジュール全体の追跡はオーバーヘッドが発生する可能性があります。そのため、できるだけ正確なモジュール名とメソッド名を使用してください。

    :::

-   **trace_error_callstack_depth** <span class='type'>Int</span>

    デフォルト値`50`

    トランザクションからエラーコールスタックを収集するときに、コールスタックの最大ライン数を指定します。このデータは、エラー統計データから照会できます。       

-   **trace_active_callstack_depth** <span class='type'>Int</span>

    デフォルト値`50`

    トランザクションからアクティブスタックを収集するときに、コールスタックの最大ライン数を指定します。

-   **trace_active_transaction_slow_time** <span class='type'>MiliSecond</span>

    デフォルト値 `3000`

    アクティブトランザクションのアークイコライザーからslow区間の応答基準を指定します。   

-   **trace_active_transaction_very_slow_time** <span class='type'>MiliSecond</span>

    デフォルト値 `8000`

    アクティブトランザクションのアークイコライザーからvery slow区間の応答基準を指定します。  

-   **trace_active_transaction_lost_time** <span class='type'>MiliSecond</span>

    デフォルト値`300000`

    トランザクションの終了を待つ制限時間です。5分以内にトランザクションが終了しない場合、トランザクション情報をこの以上収集しません。
