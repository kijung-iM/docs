ホーム面 > プロジェクト選択 > **_分析_** > **_キューブ_**

WhaTapモニタリングサービスの初期画面でプロジェクトを選択し、**_プロジェクトメニュー_**下位の**_分析_** > **_キューブ_**を選択してください。ここで、国別アクセス、ヒートマップ、トランザクション、TOPトランザクション、TPS、応答時間、リソース使用量に関する情報を提供します。

![cube-apm](https://img.whatap.io/media/images/Screenshot_2020-12-15_W_JAVA_DEMO_5490_-_Application_Monitoring19.png)

## キューブとは？

WhaTapは5分単位で作成されたパフォーマンス統計を**キューブ**と呼びます。キューブ分析は、キューブに保存された5分単位のパフォーマンスデータを利用する分析機能です。

## キューブのセレクトパネル

キューブは5分単位で保存されるため、特定の時間を選択する必要があります。パネルに表示される基準データから特定の時間を選択します。

<div style={{textAlign: 'center'}}>

![cube-select-pannel](https://img.whatap.io/media/images/cube.png)

</div>

キューブの選択パネルには、合計 5 つのインジケーターがあります。

-   **_TPS_**

-   **_応答時間_**

-   **_Tx Error件数_**

-   **_アクティブTX件数_**

-   **_同時接続ユーザー_**

キューブは、選択されたインジケーターの数値に応じて異なる色で表示されます。

-   <span class='highfg'>藍色</span>：<span class='highfg'>高い数値</span>

-   <span class='lowfg'> 水色</span>：<span class='lowfg'>low</span>

## アプリケーション選択

キューブの特定のアプリケーションからデータを選択できます。

![cube-select-apm-data](https://img.whatap.io/media/images/Screenshot_2020-12-15_W_JAVA_DEMO_5490_-_Application_Monitoring21.png)

## キューブデータ

-   **_ヒートマップ_**

    キューブ時間の経過に伴うトランザクション分布を示します。トレースの詳細分析が可能です。

    ![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、キューブ時間の**_トップスタック_**と**_ユニークスタック_**を検索できます。

    :::note

    スタック分析機能を使用できるアプリケーションは**Java**と**Python**です。 

    :::

-   **_国/地域_**

    クライアントIPに基づいて国をマッピングし、どの国からトランザクションが来ているかを示します。 円のサイズは、トランザクションの量を相対的に表示します。

-   **_TPS_**

    1秒間に処理されたトランザクション件数です。

-   **_応答時間_**

    トランザクションの平均応答時間です。

-   **_CPU #5_**

    プロジェクトのアプリケーション(エージェント)をIPでソートし、使用率の高いサーバーのCPUをチャートで表示します。

-   **_同時接続ユーザー_**

    リアルタイムのユーザーを表示します。

-   **_トップTX_**

    呼び出し件数の多い5つのトランザクションが表示されます。 トランザクション統計は、より詳細なデータを見つけることができます。

-   **_ヒープメモリ#5_**

    アプリケーション(インスタンス)でヒープメモリを大量に使用した上位5つのインスタンスが表示されます。

:::note

本製品は**MaxMind**社の**GeoLite2**のデータを使用し、それに応じてライセンスポリシーに準拠しています。**GeoLite2**は[MaxMind社ホームページ](http://www.maxmind.com)で確認できます。(This product includes GeoLite2 data created by MaxMind, available from http://www.maxmind.com)

:::
