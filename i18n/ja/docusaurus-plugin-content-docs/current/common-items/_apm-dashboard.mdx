ホーム画面 > プロジェクト選択 > **_ダッシュボード_** > **_アプリケーションダッシュボード_**

WhaTapモニタリングサービスの初期画面でプロジェクトを選択し、**_プロジェクトメニュー_**下位の**_ダッシュボード_** > **_アプリケーションダッシュボード_**を選択してください。

![アプリケーションダッシュボード](/img/apm-dashboard.png)

エージェントから収集した情報は、**_ダッシュボード_**でリアルタイムで確認できます。CPU、Memoryを除いた一般的なチャートの場合、安定したデータは青色系で表現され、問題として識別される要素は赤色系で表示されて現況を簡単に認知することができます。

## ![](/img/number-01.png)固定メニュー領域

-   ![リアルタイム通知icon](/img/ico-noti.svg)**リアルタイム通知**：最近発生したイベントが確認できます。トグルメニューをクリックしてブラウザ通知をオンまたはオフにできます。

:::note
固定メニュー領域の基本要素については、[次の文書](../main-ui-intro)を参照してください。
:::

## ![](/img/number-02.png)エージェント領域

現在のプロジェクトに関連付けられているエージェントを選択してフィルタリングできます。選択したエージェントによってウィジェット領域の指標が変更されます。

-   アクティブなエージェントの場合は青色で、非アクティブなエージェントはグレーで表示されます。
-   一番右側の![down-arrow](/img/down-arrow.svg)ボタンをクリックすると、エージェントをサーバー別、種類別にフィルタリングしたり検索することができます。

## ![](/img/number-03.png)ウィジェット（Widget）領域

ウィジェットに表示されているアイコンの機能は次のとおりです。

-   ![データマージのアイコン](/img/ico-data-merge.svg) **_データマージ_**： 該当ウィジェット項目のすべてのエージェントデータをマージし、グラフとして表示します。
-   ![個別アイコン](/img/ico-each-data.svg) **_個別_**： 各エージェントごとに分けてグラフで表示します。 ただし、プロジェクトにアプリケーションの数が多すぎると効果的ではありません。
-   ![詳細アイコン](/img/ico-detail.svg) **_詳細_**：該当ウィジェット項目のデータを詳細に照会できるモーダルウィンドウが表示されます。 **_マトリックスをもっと見る_**を選択すると、**_分析_** > **_マトリックスチャート_**メニューへ移動し、ユーザーが希望する時間帯別のデータを照会することができます。
-   ![チャート設定アイコン](/img/ico-setting.svg) **_チャート設定_**：チャートに表示する要素の設定を変更できます。 変更した事項をチャートに適用するには**_Save_**ボタンをクリックしてください。
    -   **_HorizontalLine Setting_**：チャートでユーザーが望む数値をしきい値として設定することができます。しきい値のラベル、数値、色を設定でき、追加することもできます。
    -   **_Y tick static max setting_**：Y軸に表示する最大数値を設定できます。 ユーザーが希望する数値を入力するには、**_Custom_**を選択してから数値を入力してください。

:::note
**_All APP._**アイコンが表示されているウィジェットは、アプリケーション全体に対応するデータです。
:::

### スピードメーター

![スピードメーター](/img/speedmeter.png)

スピードメーターで左から右に通る水滴アイコンは、トランザクションを意味します。アクティブトランザクションの状態を速度(<span class='normal'>Normal</span>, <span class='slow'>Slow</span>, <span class='vslow'>Very Slow</span>)で分類し、右上に個数を表示します。中央領域は進行中のトランザクションを表し、右側領域は終了したトランザクションです。

### エージェント要約

![エージェント要約](/img/agent-dashboard-summary.png)

登録したエージェントの実行状態と個数を表示します。

### **_アクティブステータス_**{#active-status}

![アクティブステータス](/img/apm-dashboard-active-status.png)

進行中のトランザクションを実行区間別に分類し、イコライザーで示します。プロジェクト内のすべてのエージェントは、5秒ごとにアクティブなトランザクション数を収集すると同時に、進捗状況を収集します。アクティブステータスは**_METHOD_**、**_SQL_**、**_HTPC_**、**_DBC_**、**_SOCKET_**に分類します。

イコライザーの色は重要度を意味します。**_DBC_**と**_SOCKET_**は、1つや2つが継続して現れたことも問題として認識します。一方、**_SQL_**と**_HTTPC_**は個数が相対的に多い場合分析が必要で、**_METHOD_**は一般的な状況なので青色です。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_ダッシュボード_** > **_アクティブトランザクション_**メニューに移動します。

-   **_METHOD_**：メソッドを実行中の状態です。
-   **_SQL_**：SQLを実行中の状態です。
-   **_HTTPC_**：外部APIを呼び出し中の状態です。
-   **_DBC_**：トランザクションがConnection Poolから新しいConnectionを取得（get）しようとしている状態です。
-   **_SOCKET_**：外部からTCP Socketを接続している状態です。

:::note
**_DBC_**または**_SOCKET_**の個数が1以上続く場合は、問題の有無を把握する必要があります。
:::

### **_アクティブトランザクション_**

![アクティトランザクション](/img/apm-dashboard-active-tx.png)

**_アクティブトランザクション_**は、進行中のトランザクションを意味します。**_アークイコライザー_**チャートとして表現されます。エージェント数だけアークが分割されます。5秒ごとに現在サーバで処理中のリクエストの数を表現して、そのリクエストがそれぞれどのくらいの時間処理されているかを知ることができます。5秒間隔の時間に感知されたリクエストは危険の有無を把握できるように色で分類します。

-   赤色(<span class='vslow'>Very Slow</span>)：8秒以上 
-   橙色(<span class='slow'>Slow</span>)：3秒~8秒
-   青色(<span class='normal'>Normal</span>)： 0秒~3秒

青色が多く表現される状況は問題になりません。これらの一部が紫色や赤色に変更されるか、推移を見守る必要があります。トラブルの状況では、進行中のトランザクション数が増加し、赤色の割合が高くなります。一方、レスポンスが非常に速いシステムの場合、処理するトランザクション数（TPS）は高くても進行中のトランザクション数は低い可能性があります。

-   左上には、進行中のトランザクションの中で最も多くのエージェント名を表示します。
-   右上のアイコンを選択してビューを切り替えることができます。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_ダッシュボード_** > **_アクティブトランザクション_**メニューに移動します。
    -   ![非アクティブエージェントを含むアイコン](/img/ico-disable-agent.svg)**_非アクティブエージェントを含む_**：非アクティブ化されたエージェントを含みます。
    -   ![アークイコライザーアイコン](/img/ico-arc-equalizer.svg) **_アークイコライザー_**：アークイコライザーでグラフを表示します。
    -   ![縦イコライザーアイコン](/img/ico-vertical-graph.svg) **_縦イコライザー_**：縦イコライザーでグラフを表示します。
-   真ん中の数字は、選択したエージェントのアクティブトランザクション数の合計です。
-   アクティブトランザクションの周りにある2本の棒は、処理量（TPS）に応じて3段階の速度でグラフの周りを回転します。

:::note
トランザクションの詳細については、[次の文書](track-transactions-intro)を参照してください。
:::

### **_ヒートマップ_**

**_ヒットマップ_**チャートは、終了したトランザクション応答時間を分布図で表現します。横軸はトランザクションの終了時間です。縦軸は実行時間です。実行時間の長いトランザクションは、分布図の上部に位置します。**_ヒットマップ_**内の領域をドラッグすると、詳細なトランザクション情報が確認できます。

:::note

-   エラーが発生せず、実行時間500ms以下のトランザクションの詳細は、URLあたり5分に1件のみ収集されます。
-   TPSや平均応答時間などの統計情報には影響しません。
-   エージェント設定により、ポリシーを調整できます。各エージェントの「設定」文書で、トランザクションに関する内容を参照してください。
:::

**_ヒートマップ_**で点色の濃さは密度を表現します。

-   正常TR：空色 → 青色 → 藍色
-   エラーTR：黄色 → 橙色 → 赤色

![ヒートマップ](/img/hitmap.png)

-   右上に位置する数字はチャート内の<span class='normal'>トランザクション件数と</span>/<span class='slow'>エラー件数</span>を意味します。
-   **_Y軸自動_**ボタンをタップすると、実行時間を示すY軸を自動調整します。
-   **_エラー_**ボタンを選択すると、エラー トランザクションのみを表示します。
-   ![上方向アイコン](/img/up-arrow.svg)または![下方向アイコン](/img/down-arrow.svg)ボタンを選択して5秒~80秒までチャートを拡大／縮小することができます。

### **_TPS_**

Tansaction Per Second（TPS）は毎秒処理したトランザクション件数を意味し、サービス性能指標の基準となります。5秒ごとに処理したトランザクションの数を1秒あたりの件数に換算してチャートで表現します。 最近10分間のTPSを示します。

![TPS](https://img.whatap.io/media/images/Screenshot_2020-12-15_W_JAVA_DEMO_5490_-_Application_Monitoring.png)

### **_本日TPS_**

本日1日のTPSを時間帯別に表示します。昨日同時間帯のTPS（グレー）と比較して確認できます。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_統計/レポート_** > **_日付別アプリケーションの現状_**メニューに移動します。

### **_本日のユーザー数_**

本日1日のユーザー数を時間帯別に表示します。昨日の同時間帯のユーザー数（グレー）と比較して確認できます。![right-arrow](/img/right-arrow.svg)ボタンをクリックすると、**_統計/レポート_** > **_日付別アプリケーションの現状_**メニューに移動します。

### **_平均応答時間_**

アプリケーションサーバーがユーザーにリクエスト結果を返すのにかかる時間をリアルタイムでモニタリングします。**_平均応答時間_**はチューニング指標として意味を持ちます。WhaTapのモニタリングサービスは、5秒間隔でトランザクションの平均応答時間を計算します。

### **_システムCPU_**

アプリケーションサーバーのCPU使用量です。リアルタイムでCPU使用量の変化推移を把握することができます。

### **_ヒープメモリ_**{#heap-memory}

各サーバーごとに使用できる最大メモリと現在のメモリを表示して、危険数値にあるサーバーを確認できます。時間によるメモリ使用量の変化をリアルタイムで見ることができます。

メモリラインチャートは通常、波打ちを続けます。アプリケーションサーバーがリクエストを処理するためにメモリを使用するとき、メモリラインチャートは増加します。GC（Garbage Collection）を通じてメモリを整理する場合は減少します。**_ヒープメモリー_**指標は平均値を使って表記します。

:::note

-   JavaとNode.jsは、ヒープメモリの総量と使用量を示しています。
-   PHPおよびPython, .Netはプロセスメモリの使用量を示しています。
:::

### **_同時接続ユーザー_**

リアルタイムブラウザのユーザー数を表示します。5秒ごとに最近5分以内にトランザクションを起こしたユーザーをカウントして表示します。ユーザーはブラウザのIPに基づいてカウントします。エージェント設定では、ユーザー区分のためにIPを使用したり、クッキーを使用することができます。

:::note

-   通常、接続ユーザーとは、現在ネットワークに接続するユーザーを意味します。しかし、ウェブシステムは非接続ネットワークを使用しているため、異なります。「接続されている」という意味ではなく、最近リクエストを転送したユーザーが測定対象になる可能性があります。
-   リアルタイム（同時接続）ユーザーとは、同じ時間帯にシステムを使用しているユーザーを意味します。リアルタイム（同時接続）ユーザーを算定したり測定する方式は様々です。
-   ユーザー数は単純に合算するのではなく、**HyperLog**で算出します。**HyperLog**は非常に少ないメモリで集合の元素個数を推定する確率的確率的データ構造を意味します。
:::
