WhaTapのアプリケーションモニタリングは、トラブルの再現なしにリアルタイムでモニタリングできます。

{@video: https://www.whatap.io/video/main-product/monitoring_step_0.mp4}

### 主な特長

-   **リアルタイムなトランザクションモニタリング**

    WhaTapは、現在進行中のトランザクションの実行状況をリアルタイムで示しています。すべてのアプリケーションの問題は、進行中のトランザクションの増加として示されます。 増加した瞬間に実行中のSQL、外部呼び出し、Methodをすぐに確認できれば、問題を最も早く認知して分析することができます。

-   **トランザクション性能の詳細分析**

    WhaTapは、すべてのトランザクションの実行履歴を収集して分析できます。 トランザクション性能はSQL、Http Callだけでなく、Methodレベルまで追跡できます。 

    <InDoc product='java,python'>

    特に**アクティブスタック**機能で煩わしいMethodトレース設定なしでMethodレベル分析が可能です。 **アクティブスタック**は、WhaTapだけの特許技術で、既存の他のサービスでは発見できなかった隠された領域の問題原因を見つけ改善ポイントを提供します。

    :::note

    但し、アクティブスタック機能はJava、Python環境のみ対応しています。

    :::

-   **MSA呼び出しパターン分析**

    MSA構造の複雑なアプリケーション間の呼び出し関係をトランザクションURL基準で呼び出し比重を統計的に視覚化して提供します。視点ごとに変化する呼び出し関係や呼び出し比重の変化を統計的に眺めることができます。 視点ごとに変化する呼び出し関係や呼び出し比重の変化を統計的に見ることができます。

-   **AIベースの応答時間分布パターン分析**

    WhaTapは、特定のパターンをマシンラーニングを通じて自動的に認識し、通知警告を発行させ、システムオペレーターが認知していないイシューまでお知らせします。（インストール型はお問い合わせください）<br/>

-   **マルチプロジェクトのトランザクション連携分析**

    MSA環境でトランザクション連携追跡機能を提供します。トランザクション全体を追跡して、どのアプリケーションで遅延が発生するかを把握できます。

</Xclude>

<InDoc product='java,python'>

-   **スタック統計の分析**

    統計分析では、リアルタイムのモニタリングやトレース情報で見つけにくいチューニングポイントを見つけることができます。 マクロ的な問題の推移もグラフと表を通じて改善対象を選別したり、特定時間帯の応答遅延現象を確認することができます。

</InDoc>

-   **事後分析（Cube）**

    時間を軸として設定し、様々な指標を一度に確認することができます。応答時間、エラー件数およびスループットなどが高い時間帯のみを特定することができ、同時時間帯の問題要素を識別したり、サービス状況を正確に見ることができます。

<InDoc product='java'>

-   **サービス過負荷時の負荷量制御**

    システムの容量を超えるサービスリクエストが流入した際には、一部のリクエストを断ったり、誤ったリクエストなら遮断したりして、サービスをより安定的に運用することができます。

-   **非同期トランザクショントレース**

    WhaTap独自のBCI（Byte Code Instrumentation）技術で非同期トランザクションの性能を効果的に追跡します。WebFlux、Reactor、Hystrixなどを使用するアプリケーションのトランザクショントレース情報を収集します。

</InDoc>
