ホーム画面 > プロジェクト選択 > **_ダッシュボード_** > **_ブラウザーモニタリングダッシュボード_**

![Browser monitoring dashboard](/img/browser-dashboard.png)

リアルタイムデータと過去のデータを照会できるダッシュボードです。 ブラウザーアプリケーションのユーザーエクスペリエンスと性能をモニタリングします。 性能指標やブラウザー、地域など、様々な側面からデータを提供することで、状態の異常を迅速に把握できます。 また、ページ読み込みイベントとAJAXイベントの性能データも提供します。

リアルタイムの統計データをチャート型ウィジェットで構成したダッシュボードです。 ページ読み込み中心の統計データとウェブブラウザーで発生するAJAXデータをリアルタイムで確認できます。

## ページ読み込みタイムライン

![Page load timeline](/img/browser-pageload-timeline.png)

ブラウザーアプリケーションの全体ページ読み込み性能の情報を提供します。 ページの読み込み時間と読み込み段階別プロセスを平均時間で示します。 これにより、ブラウザーアプリケーションの全体的な性能を把握できます。

各詳細段階の説明については、次を参考にしてください。

-   **_リダイレクト_**：ブラウザがウェブページを読み込むとき、リダイレクトの過程でかかった平均時間です。

-   **_キャッシング_**：ブラウザがウェブページを読み込み、キャッシュされたリソースを検索するのにかかった平均時間です。

-   **_DNS探索_**：ブラウザがウェブサイトを読み込みながらウェブサイトドメインを照会するのにかかった平均時間です。

-   **_TCP接続_**：ブラウザがウェブページを読み込む際、TCPヘンドシェイクの過程でかかった平均時間です。

-   **_応答待ち_**：ブラウザがウェブページを読み込みながらネットワークリクエストを送信した後、サーバーから最初のバイトが受信されるまでかかった平均時間です。

-   **_ダウンロード_**：ブラウザがウェブページを読み込み、サーバーからリソースをダウンロードするのにかかった平均時間です。

-   **_レンダリング_**：サーバーからダウンロードしたリソースを画面にレンダリングし、ページロードイベントを完了するのにかかった平均時間です。

-   **_Load Time_**：ブラウザがウェブページを完全に読み込むのにかかった平均時間です。

-   **_Front-End_**：ウェブページの初期レンダリングにかかった平均時間です。

-   **_Back-End_**：ページロードリクエストからリソースをダウンロードするのにかかった平均時間です。

## ページの読み込み時間と読み込み数

![Page load widget](/img/browser-pageload-time-num.png)

エンドユーザーがブラウザーアプリケーションを利用してウェブページにアクセスすると、各ページを読み込むのにかかる時間と頻度をリアルタイムで確認できます。 この情報により、開発者はアクセス量の多いページと読み込み速度の遅いページを特定し、速度改善が必要なページに優先順位を付けることができます。

例えば、多くのユーザーがアクセスするページの読み込み時間が長くなると、開発者はそのページの性能を改善することに集中できます。 逆に、読み込み速度が遅いページの接続量が大きくなければ、開発者はそのページの改善よりは他のページを改善することに集中した方が効率的でしょう。

:::note

全体の現況を確認するには、右上の**_ページグループ_**を選択してください。 メトリクス一覧で**_全体_**を選択してください。

:::

-   **_平均ページの読み込み時間TopN_**

    平均ページの読み込み時間をページグループごとに表示します。 水平バーチャートは、照会時間中に発生したページグループごとの読み込み時間の平均です。 上位8つまで表示されます。

-   **_ページ読み込み数TopN_**

    ページの読み込み数をページグループごとに表示します。 水平バーチャートは、照会時間中に発生したページグループごとの読み込み数の合計です。 上位8つまで表示されます。

## ページ読み込みヒートマップ

![Page load hitmap](/img/browser-pageload-hitmap.png)

**_ページロードヒートマップ_**は、時間によるページロードの応答時間を分布図チャートで表現します。

-   横軸はページ読み込みイベントの終了時間、縦軸はページ読み込みイベントの経過時間です。

-   正常ページの読み込みイベントは、空色→青色→藍色の順で表現します。

-   エラー発生ページの読み込みイベントは、黄色→オレンジ→赤色の順で表現します。 エラーページ読み込みイベントは、ページの読み込み中にブラウザーエラーが発生した場合です。

-   詳細を確認するには、チャートの特定の領域をドラッグしてください。 **_ページ読み込み詳細_**ウィンドウが表示され、ドラッグした領域の読み込みイベントに関する詳細を確認できます。 関連する詳細については、[次の文書](analyze-pageload#pageload-details)を参考にしてください。

## ページ別、メッセージ別エラー件数

![Error count](/img/browser-error-num.png)

エンドユーザーがブラウザーアプリケーションの使用中に発生したブラウザーエラーを表示します。 ページごとに発生したエラーとエラーメッセージごとに表示します。 これにより、主にエラーが発生するページとエラーに関する内容を迅速に識別できます。

-   **_ページエラー件数_**

    各ページで発生したすべてのエラー件数をページごとに表示します。 チャート上にマウスカーソルを乗せると、ページグループのエラー件数をツールチップ形式で確認できます。

-   **_エラーメッセージ別エラー件数_**

    コンソールエラーではなく、ソースエラーおよびAJAXエラーに関するメッセージを表示します。 水平バーチャートは、照会時間中に発生したエラーメッセージごとのエラー数の合計です。 上位8つまで表示されます。

## ユーザーセッション数、ユーザーセッション

![User environments](/img/browser-user-session.png)

ブラウザーアプリケーションに接続したユーザーセッションの数をリアルタイムでモニタリングできます。 これにより、ブラウザーアプリケーションの使用量を把握できます。 セッション数が急激に増えるということは、ユーザー数が増えたことを意味します。 一方、セッション数が減少すると、ユーザーがアプリケーションを使用しなくなったり、不便さを感じた可能性があります。

ユーザーセッションウィジェットと他のウィジェットを一緒に使用すると、アプリケーションの使用状況を総合的に把握できます。

-   **_ユーザーセッション数_**

    照会時間を基準に重複を除去したユーザーセッションの合計数を表示します。

-   **_ユーザーセッション_**

    5秒、5分、1時間の間隔で重複していないユーザーセッションの数を表示します。 ウィジェット右上の![詳細を見るアイコン](/img/ico-more.svg)ボタンを選択し、メトリクス変更ボタンを選択してください。 ご希望の時刻間隔を選択できます。

## 平均AJAXレスポンス時間TopN

![AJAX response time](/img/browser-ajax-response-time.png)

ブラウザーアプリケーションは、AJAXを通じてレンダリングに必要なデータをやり取りすることがよくあります。 ページ読み込み後にもAJAXが発生し続けるページでは、AJAXの読み込みがブラウザーアプリケーションのユーザーエクスペリエンスにおいて重要な指標として使用できます。 **_平均AJAXレスポンス時間TopN_**ウィジェットは、エンドユーザーがブラウザーアプリケーションを使用するときに発生するAJAXの平均レスポンス時間です。

-   AJAXリクエストpath別にチャートを提供し、リクエストホスト別、リクエストページ別、全体平均基準で選択して確認することができます。 ウィジェット右上の![詳細を見るアイコン](/img/ico-more.svg)ボタンを選択し、メトリクス変更ボタンを選択してください。 チャートデータを変更できます。

-   デフォルトでは、照会時間基準で上位10個に該当するデータが表示されます。 データ個数を変更するには、ウィジェットの右上の![もっと見るアイコン](/img/ico-more.svg)ボタンを選択した後![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択してください。 画面右上の「上位N」ボタンを選択すると、個数を変更できるオプションが表示されます。

## AJAX失敗件数

![AJAX fail count](/img/browser-ajax-fail-count.png)

AJAXエラーは、ステータスコードが400以上だったり、リクエスト自体ができなくてキャンセルされた場合を意味します。 ブラウザーアプリケーションで発生するAJAXエラー件数に対し、fetchまたはxhrタイプに分類して表示されます。

## AJAXヒートマップ

ブラウザーアプリケーションで発生したAJAXリクエストをヒートマップチャート形式で提供し、各リクエストに対する詳細情報はテーブル形式で確認できます。 各AJAXを分析して性能異常の原因を特定できます。

### ヒートマップチャート

![AJAX hitmap](/img/rum-ajax-hitmap.png)

AJAXヒートマップは、時間によるAJAXリクエストのレスポンス時間を分布図チャートで表示します。

-   横軸はAJAX終了時間、縦軸はAJAX読み込み時間です。

-   正常のAJAXは、空色→青色→藍色の順で表現します。

-   エラーAJAXは、黄色→オレンジ→赤色の順で表現します。 ステータスコードが400の場合です。 チャートでエラーだけを確認するには、ウィジェット右上の**_エラー_**ボタンを選択してください。

-   **_Y軸自動_**ボタンが活性化されていると、y軸の最大値が分布図によって自動的に変更されます。 y軸を固定するには、**_Y軸自動_**ボタンを選択して非活性化してください。

-   ![上方向アイコン](/img/up-arrow.svg)または ![下方向アイコン](/img/down-arrow.svg)ボタンを選択してチャートの位置を上・下方向に移動できます。

-   詳細を確認するには、チャートの特定の領域をドラッグしてください。 AJAX詳細情報をテーブルの形で提供します。 エラーフィルターを適用してエラーが発生したAJAXに限って確認できます。

### AJAXイベント詳細情報

![AJAX details](/img/rum-ajax-details.png)

ヒートマップチャートのセル領域をドラッグすると、詳細情報がテーブルとして提供されます。 エラーフィルターを適用してエラーが発生したAJAXに限って確認できます。 各AJAXは、次の情報を含んでいます。

-   AJAX URL

-   AJAXを要請したページURL

-   ブラウザー、機器、OS環境情報

-   エラーメッセージ情報

-   地域情報

-   AJAX詳細タイミング情報

## 地域別ページの読み込み時間と個数

![Pageload map](/img/browser-pageload-map.png)

ブラウザアプリケーションのリージョン別ページロード時間とページロード数をマップチャートで表現します。 地域ごとにウェブサイトの性能を簡単に比較できます。

-   マウスホイールを使用して地図を拡大または縮小できます。

-   特定の国の地域別地図ビューに切り替えてモニタリングするには、世界地図で国を選択してください。 現在は韓国のみ可能です。

-   国・地域別の地図に切り替えるには、ウィジェット右上の![もっと見るアイコン](/img/ico-more.svg)ボタンを選択し、メトリクスボタンを選択してください。 **_WORLD_**または国を選択できます。

-   地域別マップでは、上位の行政区画に基づいてデータを表示します。 (韓国基準：道、広域市)

## ブラウザ別ページロード

![Pageload by browser](/img/browser-pageload-by-browser.png)

ブラウザーアプリケーションごとのページ読み込み時間とページ読み込み数を提供します。 ブラウザーごとの性能を簡単に比較できます。 **_ページ読み込み時間_**または**_ページ読み込み数_**にデータを変更するには、ウィジェット右上の![もっと見るアイコン](/img/ico-more.svg)ボタンを選択し、メトリクスボタンを選択してください。

## コアウェブバイタル指標

コアウェブバイタル(Core Web Vitals)は、グーグルで定義したもので、ユーザーエクスペリエンスに影響を及ぼす様々な測定値のうち、ウェブサイトの性能に関する3つの重要な指標です。 指標は、ウェブサイトがどのように読み込まれて、ユーザーと相互作用するかによって異なります。

-   **最大視覚コンテンツの表示時間(Largest Contentful Paint)**：ウェブサイトで最大のコンテンツ要素が読み込まれる時間を意味する指標です。

-   **初回入力までの遅延時間(First Input Delay)**：ウェブサイトが読み込まれて、ユーザーが最初の入力を行うまでにかかる時間です。 

-   **累積レイアウトシフト数(Cumulative Layout Shift)**：ウェブサイトの読み込み中にレイアウトがどのくらいの頻度で変更されるかを示す指標です。

これらの測定値は、ウェブサイトの性能を評価・改善する上で重要な役割を果たします。 コア ウェブバイタル指標を理解して測定することは、ユーザーエクスペリエンスを改善し、ウェブサイトの性能を最適化する上で非常に役立ちます。

### 最大視覚コンテンツの表示時間(Largest Contentful Paint)

![LCP](/img/browser-lcp-widget.png)

各ページグループの平均最大視覚コンテンツの表示時間を表示します。 照会時間中に発生した各ページグループのうち、上位8グループの最大視覚コンテンツの表示時間の平均が水平棒チャートとして表示されます。

### 初回入力までの遅延時間(First Input Delay)

![FID](/img/browser-fid-widget.png)

各ページグループの平均初回入力までの遅延時間を表示します。 チャートの上にマウスオーバーすると、そのページグループの時間値が表示されます。

### 累積レイアウトシフト数(Cumulative Layout Shift)

![CLS](/img/browser-cls-widget.png)

各ページ グループの累積レイアウトシフト数を表示します。 チャートの上にマウスオーバーすると、そのページグループの値が表示されます。
