ブラウザーモニタリングダッシュボード機能を利用する前に、WhaTapで定義した次の用語について熟知したうえで始めてください。

## ページロード{#load-page}

ウェブアプリケーションに接続したときに、ページのコンテンツとリソースを読み込むプロセスです。 ブラウザーモニタリングサービスでは、ページ読み込み時間を次のように定義します。

-   DOMインタラクション、AJAXリクエスト、リソースレスポンスが100ms以内に来ない場合
-   「window onload」イベントが発生した場合

上記の条件を組み合わせて、最終ロード時間を決定します。

## ルーター変更

ウェブアプリケーションをSingle Page Application（SPA）として構成した場合、ページを移動するたびにページロードイベントが発生しません。 ブラウザーのHistory APIを利用してページ切り替えに関するデータを収集します。 

ルーター変更時間の定義は、ページ [ロードの最初の定義](#load-page)と同じです。

ページ切り替え後、DOMインタラクション、AJAXリクエスト、リソースレスポンスが100ms以内に来ない場合はイベントを終了します。

## ユーザーセッション

ブラウザーモニタリングサービスで **ユーザーセッション**は、ユーザーがウェブアプリケーションと相互作用を継続する行為の単位です。

ユーザーのページへのアクセスから離脱までを1つの**ユーザーセッション**として定義します。 セッションの保持期間は最大4時間であり、15分間ウェブ アプリケーションとの相互作用がない場合は、セッションを初期化します。

## ページグループ{#page-group}

ブラウザーモニタリングサービスにおいて有意義な統計グループを持つために、URL Pathを正規化して**ページグループ**という名前で提供します。 URL Path単位分析に意味がないと判断した数字、日付などは、疑問符(?)として置き換えて提供します。

例えば、_/v2/project/sms**/`12`/**dashboard/resource_board_といったURL Pathを _/v2/project/sms**/`?`/**dashboard/resource_board_といった形で提供します。

## ブラウザーエラー

ブラウザーモニタリングサービスは、ブラウザーから発生するエラーを次のように区分します。

-   「console」：開発者が定義したエラー、ブラウザーコンソールから発生するエラーです。

-   「onError」：特定のリソースの読み込みに失敗、またはスクリプトにエラーがあった場合に発生するエラーです。

-   「fetch error」：fetchを通じたHTTPリクエスト（HTTP Request）がある場合に発生するエラーです。

-   「xhr error」：xhrによるHTTPリクエスト（HTTP Request）がある場合に発生するエラーです。
