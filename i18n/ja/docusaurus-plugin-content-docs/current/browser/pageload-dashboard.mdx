**_ページロード_** ダッシュボードでは、ページロードに関連するコアウェブバイタル(Core Web Vitals、CWV) やリソース、AJAXなどの性能データをリアルタイムでモニタリングできます。 これにより、エンドユーザーのウェブアプリケーション経験に影響を及ぼす要因を把握し、改善点を導き出すのに役立ちます。

{@include: _dashboard-note.mdx}

## コアウェブバイタル関連ウィジェット

コアウェブバイタル(Core Web Vitals、CWV)は、エンドユーザーエクスペリエンスに影響を与える様々な測定値の中でグーグルが重要と考える3つのメトリクスを意味します。

![コアウェブバイタル](/img/rum-core-web-vitals.png)

:::note

グーグルの[コアウェブバイタルレポート](https://support.google.com/webmasters/answer/9205520)文書で、ページ性能がエンド ユーザーエクスペリエンスに与える影響を確認してください。

:::

### **_平均最大コンテンツレンダリング時間_**

**最大視覚コンテンツの表示時間(Largest Contentful Paint、以下LCP)**は、最初のページを読み込み始めた時点を基準に、ブラウザービューポート内にある最大の画像またはテキストブロックのレンダリング時間を報告します。 エンド ユーザーが感知するページの読み込み時間を測定できる重要なユーザー中心の指標です。 LCPが速ければ、エンドユーザーがそのページを使用できることを認識するのに役立ちます。

![平均最大コンテンツレンダリング時間](/img/rum-lcp.png)

最適化されたユーザーエクスペリエンスを提供するには、ウェブアプリケーションのLCPは**2.5秒**以下である必要があります。 **_平均最大コンテンツレンダリング時間_**ウィジェットは、ページグループまたは全体ページを基準に平均LCPを確認できます。 ページグループ基準でチャートを確認する場合、上位5ページグループの平均LCPを提供します。

### **_平均最大入力反応時間_**

**最初の入力までの遅延(First Input Delay、以下FID)**は、エンドユーザーがウェブアプリケーションのページと初めて相互作用するとき(リンクをクリックしたり、ボタンをタップしたり、カスタムJavascriptベースのコントロールを使用するなど)から、その相互作用に対するレスポンスでブラウザーが実際にイベントハンドラーの処理を開始するまでの時間を測定します。 ユーザーがレスポンスしないページと相互作用するときに感じる経験の指標です。 FIDが低くければ、ユーザーはそのページを正常に使用できると認知します。

![平均最大入力反応時間](/img/rum-pageload-fid.png)

最適化されたユーザーエクスペリエンスを提供するには、ウェブアプリケーションのFIDは**100ms**以下である必要があります。 **_平均最大入力レスポンス時間_**ウィジェットは、ページグループまたは全ページ基準で平均FIDを確認できます。 ページグループ基準でチャートを確認する場合、上位5ページグループの平均FIDを提供します。

### **_平均レイアウト変更点数_**

累積レイアウトシフト数(Cumluative Layout Shift、以下CLS)は、エンドユーザーが予期せぬレイアウト移動を経験する頻度を定量化し、視覚的な安定性を測定する際に重要なユーザー中心の指標です。 CLSが低ければ、優秀なユーザーエキスペリエンスを保証するのに役立ちます。

![平均レイアウト変更点数](/img/rum-pageload-cls.png)

最適化されたユーザーエクスペリエンスを提供するには、CLSが0.1以下を維持する必要があります。 **_平均レイアウト変更スコア_**ウィジェットは、ページグループまたは全体ページ基準で平均CLSを確認できます。 ページグループ基準でチャートを確認する場合、上位5ページグループの平均CLSを提供します。

### **_コアウェブバイタルテーブル_**

![コアウェブバイタルテーブル](/img/rum-pageload-cwv-table.png)

照会時間基準ページグループ(**_Page Group_**) 単位で平均LCP(**_Avg. LCP_**)、平均FID(**_Avg. FID_**)、平均CLS(**_Avg. CLS_**)をテーブルとして確認できます。 テーブルヘッドカラムをクリックしてカラム別にソートすることができます。

## ページロードタイミング関連ウィジェット

エンドユーザーがウェブアプリケーションに接続したときのページロード時間とロード数を通じて、ページロード時間がかかるケースと負荷が発生するページを確認できます。

### **_平均ページロード時間_** / **_ページロード数_**

![平均ページロード時間、ページロード数](/img/rum-pageload-timing.png)

エンドユーザーがウェブアプリケーションのウェブページにアクセスしたとき、リアルタイムまたは選択した時間を基準に **_平均ページのロード時間_**と **_ページロード数_**が確認できます。 ページグループ単位で読み込み時間と読み込み数を基本的に提供します。

-   ページグループ別の参照時、上位10ページグループの平均ページロード時間とページロード数を表示します。

-   ウィジェットチャートにカーソルを合わせると、ツールチップ形式でページグループと平均ページロード時間、ロード数の情報が表示されます。

### **_平均ルーター変更時間_** / **_ルーター変更回数_**

![平均ルーター変更時間、ルーター変更回数](/img/rum-pageload-router.png)

エンドユーザーがSingle Page Application(SPA)で構成されているウェブアプリケーションからページ切り替えを行った際、リアルタイムまたは選択した時間を基準に **_平均ルーター変更時間_**と **_ルーター変更回数_**をリアルタイムで確認できます。 ページグループ単位でルーターの変更時間と変更回数を基本的に提供します。

-   ページグループ別に表示すると、上位10ページグループの平均ルーターの変更時間と変更回数が表示されます。

-   ウィジェットチャートにカーソルを合わせると、ツールチップ形式でページグループと平均ルーター変更時間と変更回数の情報が表示されます。

### **_ページロードタイミングテーブル_**

![ページ読み込みタイミングテーブル](/img/rum-pageload-timing-table.png)

ページグループ(**_Page Group_**) 単位でページ読み込み数(**_Load count_**)、平均読み込み時間(**_Avg. Load Time_**)、平均バックエンド処理時間(**_Avg. Backend Time_**)、平均フロントエンド処理時間(**_Avg. Frontend Time_**)をテーブルとして確認することができます。 テーブルヘッドカラムをクリックしてカラム別にソートすることができます。

## リソース関連ウィジェット

ページの読み込みに影響を与えるリソースを識別できます。 リソースの読み込み時間は、800msを超えない方がいいです。

### 平均リソースレスポンス時間 / リソース件数

![平均リソースレスポンス時間、リソース件数](/img/rum-pageload-resource.png)

ページの読み込み時に発生した平均リソースレスポンス時間とリソース件数をチャートとして提供します。 ウィジェット右上の **_リソースパス_** ボタンを選択して **_リソースホスト_**、**_全体_** リソースを基準に確認できます。

-   **_リソースパス_**、**_リソースホスト_** を条件に上位10に該当する平均リソースレスポンス時間およびリソース件数を表示します。

-   ウィジェットチャートにカーソルと合わせると、ツールチップ形式で平均リソースレスポンス時間とリソース件数情報が表示されます。

### リソーステーブル

![リソーステーブル](/img/rum-pageload-resource-table.png)

**_ページ読み込み - リソーステーブル_**ウィジェットでは、リソースURLのPath基準とリクエストしたページグループ基準をテーブルとして確認できます。 ウィジェット右上の**_リソースPath_**ボタンを選択して**_ページグループ_**基準に切り替えてデータを照会することができます。

## AJAX関連ウィジェット

ページの読み込みに影響を与えるAJAXを識別できます。 レンダリングに必要なデータをAJAXを通じて提供される場合、レスポンスが遅いと画面のレンダリングが長くかかることがあります。

### 平均AJAXレスポンス時間 / AJAX件数

![AJAXレスポンス時間、AJAX件数](/img/rum-pageload-ajax.png)

ページの読み込み時に発生した平均AJAXレスポンス時間およびAJAX件数をチャートとして提供します。 ウィジェット右上の**_AJAX Path_** ボタンを選択し **_AJAX Host_**、**_全体_** AJAX基準に切り替えてデータを照会することができます。

-   **_AJAX Path_**、**_AJAX Host_** 条件で参照時、上位10に該当する平均AJAXレスポンス時間とAJAX件数を表示します。

-   ウィジェットチャートにカーソルを合わせると、ツールチップの形式で平均AJAXレスポンス時間とAJAX件数の情報を表示します。

### AJAXテーブル

![AJAXテーブル](/img/rum-pageload-ajax-table.png)

**_ページ読み込み - AJAXテーブル_**ウィジェットでは、AJAX URLのPath基準とリクエストしたページグループ基準をテーブルとして確認できます。 ウィジェット右上の**_AJAX Path_**ボタンを選択して**_ページグループ_**基準に切り替えてデータを照会することができます。
