ホーム画面 > プロジェクト選択 > **_分析_** > **_エラー追跡_**

ブラウザーモニタリングは、ブラウザーのフロントエンド(Front-End) エラーを収集します。 エラーメッセージおよびエラースタック情報、ブラウザー、OS、デバイス、ページのURL情報が含まれています。

## エラー探索

時間の経過とともに発生したブラウザーエラーを照会できます。

![エラー追跡](/img/rum-tracking-error.png)

1.  ![数字1](/img/number-01.png)ご希望の時間を設定してください。 最大1日前まで設定できます。

2.  チャートに表示された ![数字 2](/img/number-02.png)棒グラフから1つを選択してください。 

    ![数字 3](/img/number-03.png)時間一覧から10分単位で収集したデータの中からエラーが発生した詳細時間を選択できます。

3.  選択した時点で発生したブラウザーエラーをチャートの下で一覧として確認できます。

    ![数字 5](/img/number-05.png)テキスト入力ボックスに特定のキーワードを入力すると、一覧から一致するテキストをハイライト表示します。

4.  エラーの詳細を確認するには、一番左の ![数字 4](/img/number-04.png) ![詳細を見るアイコン](/img/ic-caret-right.svg)ボタンを選択してください。

## エラー詳細情報を確認する

ブラウザーエラーには、次の情報が含まれています。

-   **_errorType_**：エラーのタイプを意味します。 AJAXエラー、コンソールエラー、Syntaxエラーを含むソースエラーがあります。

-   **_pageLocation_**：ユーザーが訪問したページURLです。

-   **_title_**：ユーザーが訪問したページのタイトル情報です。

-   **_url_**： 

-   **_browser_**：ユーザーが利用したブラウザーの種類です。

-   **_borwserVer_**：ユーザーが利用したブラウザーのバージョンです。

-   **_os_**：ユーザーが利用した機器のOSの種類です。

-   **_osVer_**：ユーザーが利用した機器のOSバージョンです。

-   **_device_**：ユーザーが利用した機器の種類です。

-   **_host_**：ユーザーが訪問したサーバーのドメイン名です。

-   **_userAgent_**：ユーザーが利用したブラウザーから受け取った「User-Agent」文字列です。

-   **_message_**：エラーメッセージです。

-   **\*stack**：エラースタック情報です。 エラースタックには、基本的に難読化されたファイル名とライン、カラム情報が含まれます。

## ソースマップのアップロードおよびエラーコードの提供

配布されたフロントエンドコードが難読化または圧縮された場合、収集されたブラウザーエラーのスタック情報を実際のコードレベルで提供することは困難です。 フロントエンドコードを配布する時点でビルドされたsource-mapを通じてブラウザーエラーが発生したファイル、コードラインにアクセスできます。

1.  **_stack_** 情報から ![アップロードアイコン](/img/ico-upload.svg)ボタンをクリックすると、そのコードスタックに対するソースマップを読み込むことができます。 

    ![](/img/rum-sourcemap-upload.png)

2.  難読化されたスタック情報とJavaScriptのソースマップ情報を比較して、実際にエラーが発生したコードを確認できます。

    ![](/img/rum-sourcemap-descrypt.png)

## エラー情報をフィルタリングして探索する

ブラウザエラーに含まれた情報を基準にフィルタリングし、エラー情報を照会することができます。

![](/img/rum-tracking-error-filtering.png)

1.  画面の左パネルでフィルタリングしたいブラウザーエラー情報の項目を選択してください。 

2.  下位一覧が表示されたら、ご希望の項目を選択してください。 一覧が多すぎる場合は、検索することもできます。 

選択した項目に該当するブラウザーエラーの一覧を表示します。

## テーブルのカラム設定

### テーブルカラムの表示・非表示

ブラウザーエラー一覧テーブルのカラム ヘッダー項目を表示・非表示にすることができます。

1.  テーブル右上の ![設定アイコン](/img/ico-setting.svg)ボタンを選択してください。

2.  ドロップダウンメニューから **_カラム設定_**を選択してください。

3.  **_カラム設定_** ウィンドウが表示されたら **_カラム追加_** セクションでお探しの項目を選択するか、選択を解除してください。

4.  すべての設定を完了して **_確認_** ボタンを選択してください。

設定した内容がテーブルに適用されます。

### テーブルカラムの順序付け

ブラウザーエラー一覧のテーブルでカラム ヘッダーの順序を変更できます。

1.  テーブル右上の ![設定アイコン](/img/ico-setting.svg)ボタンを選択してください。

2.  ドロップダウンメニューから **_カラム設定_**を選択してください。

3.  **_カラム設定_** ウィンドウが表示されたら **_カラム順序設定_** セクションでお探しの項目をドラッグして順序を変更してください。

4.  すべての設定を完了して **_確認_** ボタンを選択してください。

設定した内容がテーブルに適用されます。

## テーブルコンテンツの表示設定

ブラウザーエラー一覧に表示するコンテンツの内容を設定できます。 多すぎる情報が表示されて見づらい場合や必要な内容を追加したい場合に利用できます。

1.  テーブル右上の ![設定アイコン](/img/ico-setting.svg)ボタンを選択してください。

2.  ドロップダウンメニューから**_テーブルコンテンツ設定_**を選択してください。

3.  次のオプションを設定してください。

    -   **_コンテンツ表示有無_**：テーブル一覧に表示するコンテンツの表示有無を選択してください。

    -   **_タグ管理_**：テーブル一覧に表示したいタグを選択できます。 見えないように設定したり、手順を変更したりできます。

4.  すべての設定を完了して **_確認_** ボタンを選択してください。

設定した内容がテーブルに適用されます。
