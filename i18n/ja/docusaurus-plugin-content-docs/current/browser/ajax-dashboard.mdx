ホーム画面 > プロジェクト選択 > **_ダッシュボード_** > **_AJAX_**

AJAXとは、非同期ジャバスクリプトとXML（Asynchronous JavaScript And XML）です。 서버와 통신하기 위해 `XMLHttpRequest` 객체를 사용하는 것을 말합니다. 

AJAXは、リソース データのうち動的な画面を構成する上で重要なデータロード方式です。 AJAXが正常に呼び出されない場合、エンドユーザーに画面が正常に表示されません。 この時に収集する情報は、ユーザーエクスペリエンスを損なう要素を把握するのに役立ちます。

{@include: _dashboard-note.mdx}

## **_AJAX件数_**

![AJAX件数](/img/rum-ajax-count.png)

リアルタイムでAJAXのロード数を確認することができます。

-   ウィジェット右上の ![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択し、時間帯を設定してAJAXのロード数を照会することができます。 
-   データ照会基準を **_AJAX Host_** または **_全体_** AJAXに変更するにはウィジェット右上の **_AJAX Path_** ボタンを選択してください。
-   **_AJAX Path_**、**_AJAX Host_** を基準にAJAXロード数の照会時、上位10個に該当するAJAX件数を提供します。
-   ウィジェットチャートにマウスオーバーすると、ツールチップの形でAJAX件数情報を提供します。

## **_平均AJAXレスポンス時間_**

![平均AJAXレスポンス時間](/img/rum-ajax-response-time.png)

リアルタイムでAJAXレスポンス時間を確認することができます。 特定のPath、Host単位で平均AJAXレスポンス時間が長くかかるAJAXを識別できます。

-   ウィジェット右上の ![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択し、時間帯を設定してAJAXレスポンス時間を照会することができます。 
-   データ照会基準を **_AJAX Host_** または **_全体_** AJAXに変更するにはウィジェット右上の **_AJAX Path_** ボタンを選択してください。
-   **_AJAX Path_**、**_AJAX Host_** を基準にAJAXロード数の照会時、上位10個に該当するAJAXレスポンス時間を提供します。

## **_ステータス別のAJAX_**

![ステータス別のAJAX](/img/rum-ajax-status.png)

時間帯のAJAXに関連したHTTPステータスコードを確認することができます。

![ステータス別のAJAX](/img/rum-ajax-status-count.png)

AJAXと関連したHTTPステータスコードを照会した、時間基準の累積件数を確認することができます。

AJAXに関するステータスは次のとおりです。 次の基準で、WebアプリケーションでAJAXが正常に動作するか確認してください。

-   **2xx**（成功）: リクエストの受け取りに成功・認識し、受け入れました。
-   **3xx**（リダイレクション）：リクエスト完了のために追加の措置が必要です。
-   **4xx**（クライアントエラー）: リクエストの文法が間違っているか、リクエストを処理できません。
-   **5xx**（サーバーエラー）: サーバーが明らかに有効なリクエストを満たすことに失敗しました。

:::note
HTTPステータスコードの詳細については、[次の文書](https://www.whatap.io/ko/blog/40/)を参考にしてください。
:::

## **_AJAXエラー件数_**

![AJAXエラー件数](/img/rum-ajax-error-count.png)

リアルタイムでAJAXに関連したエラー件数を確認することができます。

-   ウィジェット右上の ![詳細を見るアイコン](/img/ico-detail-mode.svg)ボタンを選択し、時間帯を設定してAJAXのエラーロード件数を照会することができます。 
-   データ照会基準を **_AJAX Host_** または**_ページグループ_**に変更するにはウィジェット右上の **_AJAX Path_** ボタンを選択してください。
-   **_ページグループ_**を基準に照会する際、ウィジェットチャートにマウスオーバーすると、ツールチップを通じて特定のページグループから発生したAJAXエラーを識別できます。
